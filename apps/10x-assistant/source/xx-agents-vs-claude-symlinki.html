AGENTS.md vs CLAUDE.md - jak ogarnąć wiele konwencji reguł i instrukcji?

Sporo osób używa różnych narzędzi AI do kodowania: Claude Code, Cursor, GitHub Copilot, Gemini CLI.

Problem? Każde narzędzie chce innego pliku z kontekstem projektu.

Rozwiązań się namnożyło jak grzybów po deszczu, a wraz z nimi różne “standardy”:

  • Claude Code → CLAUDE.md

  • Gemini CLI → GEMINI.md

  • Cursor → .cursor/rules

  • Cline → .clinerules/rules

  • Roo → .roorules/rules

  • OpenAI Codex → AGENTS.md

  • AMP → AGENT.md

Masz projekt i chcesz używać Cursor rano, Claude Code po południu, a Copilot wieczorem? Zostaje utrzymywać 3,4,5 plików jednocześnie….

może, że jest na to inny sposób? :)

Rozwiązanie: Symlinki

Zamiast kopiować zawartość między plikami, zrób symlink - skrót który wskazuje na ten sam plik.

macOS/Linux:

# Utwórz AGENTS.md jako główny plik
  touch AGENTS.md

  # Stwórz symlinki dla narzędzi które jeszcze nie wspierają standardu
  ln -s AGENTS.md CLAUDE.md
  ln -s AGENTS.md GEMINI.md

  # Teraz masz jeden plik, trzy nazwy
  # Edytujesz AGENTS.md → zmiany widać wszędzie

Windows (CMD jako administrator):

mklink CLAUDE.md AGENTS.md
  mklink GEMINI.md AGENTS.md

Windows (PowerShell jako administrator):

New-Item -ItemType SymbolicLink -Path "CLAUDE.md" -Target "AGENTS.md"
  New-Item -ItemType SymbolicLink -Path "GEMINI.md" -Target "AGENTS.md"

Voila! Jak to zrobiłeś, to zedytujesz AGENTS.md, Claude Code automatycznie widzi zmiany przez CLAUDE.md.

TL;DR

  1. Stwórz AGENTS.md jako główny plik

  2. Zrób symlinki do CLAUDE.md, GEMINI.md dla narzędzi które jeszcze nie wspierają standardu

  3. Commituj AGENTS.md do repo

  4. Edytujesz jeden plik → wszystkie AI go widzą

Jeden plik zamiast trzech/czterech. Zero duplikacji. Mniej bólu głowy.

Używacie innych rozwiązań do zarządzania plikami kontekstu? Wrzucajcie w komentarzach.

PS. Zdaję sobie sprawę, że można inaczej określić/sprecyzować instrukcję per narzędzie, ale to jest sposób dla ludzi, którzy chcą zarządzać jedną dokumentacją .md i nie utrzymywać paru w projektem. Zostaw ❤️ jak ci się podoba.
