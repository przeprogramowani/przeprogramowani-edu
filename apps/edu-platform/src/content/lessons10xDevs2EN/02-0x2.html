Github Copilot

Copilot is, for many programmers, the first step onto the path of a 10xDev, i.e., a developer collaborating with AI. It is a plugin that tries to fit all the most popular ecosystems and programming environments (how it works in practice - a bit more on that later):

We will introduce the three basic scenarios of collaborating with AI using this tool. They will be:

  • Inline Edits

  • Chat with AI

  • Multi-file Edits

Additionally, while preparing for the next lessons, you will also learn about:

  • Instructions for AI

  • Terminal commands

Copilot benefits from availability and a strong brand, but in practice it turns out that collaboration with this tool – especially at the whole company level – may leave much to be desired.

It is easy to notice that Copilot roadmaps for popular editors are developed at different speeds. Priority is given to the VS Code editor, then the original Visual Studio, later JetBrains, while support for XCode... is an experiment in the pre-release version.

Here you can see a strong dependency on the specific stack you will be working with. In this context, iOS developers working in XCode cannot expect the same “DX” as others (similarly as in JetBrains compared to Visual Studio).

The second issue is the matter of models. Until recently, “Copilot for JetBrains” used GPT-4 released a year ago, which compared to Claude 3.7 Sonnet is a previous generation model. Moreover, we could not change this – today there is a so-called “model selector” available, but community ratings for the plugin itself speak for themselves (as of April 2025):

And finally, the speed of development issue. Copilot tries to be the most stable of the stable plugins, which unfortunately translates into a slower update pace. The Claude 3.5 Sonnet recommended in the previous lesson appeared on the market in June 2024, and in Copilot four months later. The “3.7” version was fortunately deployed faster, but in other areas (including indexing external sources of knowledge, agent mode, or support for DeepSeek models) there are significant lags.

Cursor and Windsurf, i.e. VS Code with native AI

The creators of two other tools - Cursor and Windsurf - decided to break away from the limitations of “editor plugins” by creating their own development environment. Both Cursor and Windsurf are based on VS Code, but these are independent, alternative applications that you run on your own computer.

Cursor - https://cursor.com/

If you work in a full-stack JavaScript ecosystem (including Node and TypeScript), and you also know VS Code, Cursor will be a significant step forward compared to Copilot and our recommendation for the main part of the 10xDevs training.

Cursor offers, among others:

  • easy access to the best models such as Claude 3.7 Sonnet

  • a working (!) and available for many months Agent (Cursor Composer)

  • intelligent code completion (Cursor Tab)

  • models provided in the OpenRouter.ai service

  • web information search (@Web) and indexing of new knowledge sources (@Docs)

You can start using Cursor for free - after registration, a 2-week Trial of the Pro plan is activated. After this period, the subscription will cost $20/month - at this price you get unlimited code auto-completion, 500 “fast” queries to the best models, and unlimited slower queries once you use up that limit (then you can switch to a pay-as-you-go plan).

Cursor also supports using your own model API keys, but then some features (like Agent or Cursor Tab) are not available - a solution may be integration with Open Source plugins like Aider or Cline, but the entire integration becomes quite complex and is not the easiest configuration to handle.

Refreshing the index

In projects where the content and location of files change quickly, the index is not always refreshed in real-time, and as a result, the precision of AI model responses decreases. When you notice such a problem, refresh (or completely delete) the built index in the project. This process will not remove data from the repository but only the “hints for AI,” which will be recreated shortly.

Cursor provides this option in the “Features” section in settings:

For GitHub Copilot users, this option is available in the command field (VS Code) - rebuild the local (if you work in VS Code) or remote index (if you use Copilot on GitHub.com).

Additionally, Copilot notes that in freshly opened projects, response effectiveness may increase over time (a few minutes after start) due to the completion of the file indexing process.

Windsurf - https://codeium.com/windsurf

Windsurf is an alternative to Cursor, also developed as an independent project based on the core functionalities of VS Code. The creators previously focused on the Codeium plugin, but now the product and marketing effort is moving towards a full environment for developers.

The philosophy of Windsurf is somewhat different - while Cursor extends VS Code's capabilities with AI features in many different directions, Windsurf mainly focuses on the Agent mode named Cascade. It is also an editor targeted at large enterprises - both through the foundation of a more stable infrastructure, created well before the launch of the best AI models, professional communication of changes (important for many CTOs), and Codeium backend deployments in on-prem and dedicated clouds (more about this on the blog).

The subscription costs $15/month (5$ cheaper than Cursor), however Windsurf does not differentiate queries into “counted fast” and “unlimited slow” - for the mentioned amount you simply get a budget of 500 queries to premium models and 1500 actions with tools like Web Search, and after using the limit, the account must be recharged. In agent mode, with network searching enabled (each query equals 1 action), these limits run out really quickly, as many power users point out.

Other significant differences compared to Cursor? Currently, no support for the OpenRouter service (limited number of available models) and no access to symbols in Inline Edit mode.

Pay-as-you-go - Cline, Aider and Claude Code

Many developers, especially at the initial stage of working with AI, do not want to enter paid monthly subscriptions.

A solution is the pay-as-you-go model, which allows generating your own API key for the platform and paying only for actually used resources, without long-term commitments. Supported by, among others, Cline (VS Code plugin), Aider (video about it below) and an early version of Claude Code (Anthropic’s terminal agent).

Each of these tools needs an API key for the preferred model provider (Claude Code is limited to Claude Sonnet models, but they can be hosted on Amazon Bedrock and Google Vertex AI - then you generate the key on that platform).

  • OpenAI (GPT-4o, GPT-4.5 and o1/o3-mini models) - https://platform.openai.com/

  • Anthropic (Claude Sonnet models) - https://console.anthropic.com/

  • Google (Gemini models) - https://aistudio.google.com/

  • OpenRouter - https://openrouter.ai/

Below, we present the Aider solution in more detail. The reason is simple - it's a really efficient terminal agent that you can integrate with JetBrains family editors.

Choose the optimal environment for AI collaboration

We hope that our development of tools for 10xDev will help you configure an AI collaboration environment tailored to your habits and preferences. To complete the main part of the training, you will need at least one of the editors or plugins presented here.

The minimum setup (below) is met, among others, by Copilot, Cursor, and JetBrains AI Assistant:

  • Access to Claude 3.5/3.7 Sonnet and o3-mini models

  • The ability to configure Rules for AI

  • For easy AI collaboration - "Chat with AI" / "Inline Edit" features

Our recommendation in the VS Code ecosystem is:

1) Cursor (offers a 2-week premium account trial to start)

2) Copilot / Copilot Insiders (definitely with AI rules - covered in upcoming lessons)

3) Cline (requires own API key)

In the JetBrains ecosystem, you can choose:

1) AI Assistant subscription (+ sign up for the Junie waitlist)

2) Copilot subscription combined with Aider in the terminal

If you have any questions, we are waiting for comments and suggestions - good luck!
