A setup that's enough to get started

You don't need a perfect environment for Module 1. You need an environment that allows you to actually work with an AI agent, not just chat with a chatbot in the browser.

At a minimum, this means one selected agent tool: Cursor, Claude Code, Codex, or another alternative that can work with the repository, read files, suggest changes, and help verify the result. If you still don't know which environment to choose, return to the lesson Agent in IDE, Terminal, or Cloud. There we distinguish the ergonomics of IDE, terminal, and agent-native tools without turning it into a tool religion.

The second thing is the billing model. From May 18, it is worth having an active subscription to the selected tool or consciously using the API. The API offers flexibility but can quickly cost more than 20 dollars per month. If the topic of models, costs, and local alternatives is unclear to you, go back to the lesson Recommended models and how to stay up to date.

At the start, we do not recommend local models as the main working environment. It’s a good topic for experiments, but a poor default for the course if you don’t have a strong machine, a lot of patience, and readiness for lower quality in agent tasks. In Module 1, it’s better to start with a cloud tool that works stably than with a local setup that requires performance (of the machine and the agent itself) during the first two evenings.
