Claude Code is your personal Programming Agent, equipped with top models like Opus and Sonnet, controlled by a quality harness - a layer of tools, security, and call control.

It is this famous harness, i.e. the integration layer of the models themselves, including with your file system or external resources, that made Claude Code quickly rise to become a top AI tool for developers – while access to the same models was previously offered by other tools such as Cursor, few alternatives utilized the full potential of these solutions – whether at the level of invoking the right tools, using the terminal, correcting its actions, or efficiently searching files and maintaining up-to-date knowledge about the project.

If the difference between the model, the Agent, and the harness is not clear to you, go back to one of the previous lessons where we explain this with examples. However, if you have already completed that lesson, let's now focus on the basics of working with Claude Code.

Claude Code - Operational Basics

If you are using another tool

Not every agent tool works in the terminal like Claude Code, but you should evaluate each through a similar set of questions. The interface may be different, but the agent's effectiveness, work cost, and security still remain your responsibility.

Start with permissions. Check when the tool asks for consent to edit files, run commands, read data outside the project directory, use the network, and connect external integrations. If there is a mode without asking for consent, treat it as an option for controlled experiments, not the default way of working on an important repository. If you notice that the selected Agent + Model configuration goes beyond the workspace, consider a different environment.

Then review the operational commands. Look for equivalents of commands for help, model selection, context checking, change preview, session clearing, conversation compacting, and cost control. These are not decorative features for power users, but rather basic control methods.

The next point is project memory. Check if the tool has a configuration file similar to CLAUDE.md, project rules, or constant instructions added to each session.

Support for conventions becoming standards will also be important, including AGENTS.md and .agents/skills.

  • AGENTS.md - standardization of Agent onboarding in the project
  • .agents/skills - new possibilities and knowledge for Agents

If you are looking for an alternative to Claude Code, today we recommend two directions. The first is Codex with GPT-5.* models. The second is OpenCode combined with leading open models, such as Kimi K2.6 or GLM-5.1. This is usually a more economical option.

During the training, you will also get to know our benchmark of tools and models, which justifies these recommendations. Without a benchmark, it is easy to compare tools based on start screens and marketing, and that is not the most precise method of working.

Don't forget about privacy policies - check if and when the code goes into the model training process as well as what limits are available in the subscription, i.e., when you pay a fixed amount and when according to the token usage via the API.

At 10xDevs we will further unpack this topic, because an agent without budget control can be very persuasive... and very expensive.
