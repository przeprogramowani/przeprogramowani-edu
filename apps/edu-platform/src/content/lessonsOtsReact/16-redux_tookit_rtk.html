Redux Toolkit (RTK) to zestaw narzędzi, które upraszczają pracę z Reduxem, oferując uproszczoną konfigurację store, tworzenie reducerów oraz zarządzanie stanem aplikacji. W połączeniu z TypeScript, RTK umożliwia tworzenie silnie typowanych aplikacji, co zwiększa bezpieczeństwo kodu, poprawia czytelność oraz ułatwia utrzymanie aplikacji. W tej lekcji omówimy, jak efektywnie stosować TypeScript w Redux Toolkit, korzystając z kluczowych koncepcji i najlepszych praktyk.

Dokumentacja Redux Tookit posiada rozbudowany poradnik jak korzystać z TypeScripta w tej bibliotece. Ta lekcja to pigułka wiedzy, dzięki której poznasz najważniejsze elementy integracji, niezbędne do efektywnej pracy.

Definiowanie typów RootState i Dispatch

Aby zapewnić silne typowanie w aplikacji e-commerce, zaczynamy od zdefiniowania typów RootState oraz AppDispatch. Te typy będą wykorzystywane w całej aplikacji do typowania stanu oraz funkcji dispatch.

// src/app/store.ts
import { configureStore } from '@reduxjs/toolkit'
import usersReducer from '../features/users/usersSlice'
import productsReducer from '../features/products/productsSlice'

export const store = configureStore({
  reducer: {
    users: usersReducer,
    products: productsReducer,
  },
})


export type RootState = ReturnType<typeof store.getState> // Wywnioskowany typ: { users: UsersState, products: ProductsState }
export type AppDispatch = typeof store.dispatch

  • configureStore: Funkcja z Redux Toolkit, która upraszcza konfigurację store, automatycznie łącząc middleware oraz umożliwiając łatwe dodawanie reducerów.

  • usersReducer i productsReducer: Reducery odpowiedzialne za zarządzanie stanem użytkowników oraz produktów w aplikacji e-commerce.

  • RootState: Typ reprezentujący cały stan store, wywnioskowany za pomocą ReturnType<typeof store.getState>. Dzięki temu RootState zawiera struktury UsersState oraz ProductsState.

  • AppDispatch: Typ reprezentujący funkcję dispatch, uzyskany przez typeof store.dispatch. Umożliwia to poprawne typowanie akcji dispatchowanych w aplikacji.

Tworzenie typowanych hooków

Zamiast importować typy RootState i AppDispatch w każdym komponencie, tworzymy typowane wersje hooków useDispatch i useSelector. Ułatwia to korzystanie z nich w całej aplikacji i zapobiega błędom związanym z typowaniem.

// src/app/hooks.ts
import { TypedUseSelectorHook, useDispatch, useSelector } from 'react-redux'
import type { RootState, AppDispatch } from './store'

// Używaj w całej aplikacji zamiast standardowych `useDispatch` i `useSelector`
export const useAppDispatch: () => AppDispatch = useDispatch
export const useAppSelector: TypedUseSelectorHook<RootState> = useSelector

  • TypedUseSelectorHook: Typ z react-redux, który pozwala na stworzenie typowanej wersji hooka useSelector.

  • useAppDispatch: Typowany hook do dispatchowania akcji, zapewniający poprawne typowanie thunków i innych middleware.

  • useAppSelector: Typowany hook do wyciągania stanu z store, eliminujący potrzebę wielokrotnego deklarowania typu stanu.

Definiowanie typów state i akcji w slice

Każdy slice powinien mieć zdefiniowany typ dla swojego stanu początkowego. Dzięki temu createSlice będzie mógł poprawnie wywnioskować typ stanu w reducerach.

Przykład Slice: usersSlice.ts

// src/features/users/usersSlice.ts
import { createSlice, PayloadAction } from '@reduxjs/toolkit'

// Definiowanie typu użytkownika
export interface User {
  id: number
  name: string
  email: string
}

// Definiowanie typu stanu
export interface UsersState {
  status: 'idle' | 'loading' | 'succeeded' | 'failed'
  data: User[] // Lista użytkowników
  error: string | null
} 

// Stan początkowy
const initialState = {
  status: 'idle',
  data: [],
  error: null,
} satisfies UsersState as UserState

export const usersSlice = createSlice({
  name: 'users',
  initialState,
  reducers: {
    setUsers: (state, action: PayloadAction<User[]>) => {
      state.data = action.payload
      state.status = 'succeeded'
    },
    setError: (state, action: PayloadAction<string>) => {
      state.error = action.payload
      state.status = 'failed'
    },
  },
})

export const { setUsers, setError } = usersSlice.actions
export default usersSlice.reducer

Wyjaśnienie:

  • User: Interfejs definiujący strukturę obiektu użytkownika.

  • UsersState: Interfejs definiujący strukturę stanu slice dla użytkowników, zawierający status operacji, listę użytkowników oraz ewentualny błąd.

  • initialState: Obiekt inicjalizujący stan zgodnie z UsersState. Ważne: warto zastosować satisfies połączone z as zamiast adnotacji typ. Unikniemy wtedy występujących sporadycznie problemów z zawężaniem akceptowalnego stanu do stanu startowego.

  • createSlice: Funkcja z Redux Toolkit, która upraszcza tworzenie slice'a, automatycznie generując akcje i reducera.

  • reducers: Zestaw funkcji redukujących stan w odpowiedzi na akcje. Każda funkcja jest typowana przy użyciu PayloadAction<T>, gdzie T określa typ przesyłanego payloadu.

Generowanie typów akcji dla slice'ów

Redux Toolkit automatycznie generuje typy akcji, łącząc nazwę slice'a z nazwą reduktora, tworząc literały stringów typu 'sliceName/actionName'. 

// src/features/products/productsSlice.ts
import { createSlice, PayloadAction } from '@reduxjs/toolkit'

export interface Product {
  id: number
  name: string
  price: number
  description: string
}

export interface ProductsState {
  items: Product[]
  status: 'idle' | 'loading' | 'succeeded' | 'failed'
  error: string | null
}

const initialState: ProductsState = {
  items: [],
  status: 'idle',
  error: null,
}

export const productsSlice = createSlice({
  name: 'products',
  initialState,
  reducers: {
    setProducts: (state, action: PayloadAction<Product[]>) => { // action.type: 'products/setProducts'
      state.items = action.payload
      state.status = 'succeeded'
    },
    setError: (state, action: PayloadAction<string>) => { // action.type: 'products/setError'
      state.error = action.payload
      state.status = 'failed'
    },
  },
})

export const { setProducts, setError } = productsSlice.actions
export default productsSlice.reducer

Można również używać metody match. match działa jako type guard, umożliwiając zawężenie do konkretnego typu akcji generowanego przez slice. Dzięki temu TypeScript rozpoznaje typ action jako PayloadAction<Product[]>:

// src/middleware/customMiddleware.ts
import { Middleware, MiddlewareAPI, Dispatch, Action } from '@reduxjs/toolkit'
import { productsSlice } from '../features/products/productsSlice'

const customMiddleware: Middleware = (api: MiddlewareAPI<Dispatch, any>) => (next) => (action: Action) => {
  if (productsSlice.actions.setProducts.match(action)) {
    // `action` jest teraz typowane jako `PayloadAction<Product[]>`
    console.log('Products set:', action.payload)
  }
  return next(action)
}

export default customMiddleware

Praca z extraReducers

extraReducers służą do reagowania na akcje zdefiniowane w innych slice’ach lub asynchronicznych thunkach. Definiowanie extraReducers może być skomplikowane pod względem typowania. Redux Toolkit zaleca użycie podejścia z callbackiem buildera, co zapewnia lepsze bezpieczeństwo typów na poziomie obsługi konkretnej akcji:

// src/features/products/productsSlice.ts
import { createSlice, createAsyncThunk, PayloadAction } from '@reduxjs/toolkit'
import type { RootState } from '../../app/store'

export interface Product {
  id: number
  name: string
  price: number
  description: string
}

export interface ProductsState {
  items: Product[]
  status: 'idle' | 'pending' | 'succeeded' | 'failed'
  error: string | null
}

const initialState: ProductsState = {
  items: [],
  status: 'idle',
  error: null,
}

export const fetchProductById = createAsyncThunk<Product, number>(
  'products/fetchById',
  async (productId: number) => {
    const response = await fetch(`https://fakestoreapi.com/products/${productId}`)
    if (!response.ok) {
      throw new Error('Network response was not ok')
    }
    const data = (await response.json()) as Product
    return data
  }
)

export const productsSlice = createSlice({
  name: 'products',
  initialState,
  reducers: {
    addProduct: (state, action: PayloadAction<Product>) => {
      state.items.push(action.payload)
    },
  },
  extraReducers: (builder) => {
    builder
      .addCase(fetchProductById.pending, (state) => {
        state.status = 'pending'
      })
      .addCase(fetchProductById.fulfilled, (state, action: PayloadAction<Product>) => {
        state.status = 'succeeded'
        state.items.push(action.payload)
      })
      .addCase(fetchProductById.rejected, (state, action) => {
        state.status = 'failed'
        state.error = action.error.message || 'Failed to fetch product'
      })
  },
})

Wyjaśnienie:

  • builder.addCase: Metoda addCase umożliwia dodanie obsługi konkretnych akcji, takich jak pending, fulfilled i rejected generowanych przez createAsyncThunk.

  • Typowanie akcji: Dzięki builderowi, TypeScript poprawnie typuje state oraz action dla każdej obsługiwanej akcji, co zapewnia bezpieczeństwo typów i eliminuje błędy.

Podsumowanie

Wykorzystanie TypeScript w Redux Toolkit znacząco poprawia czytelność kodu i bezpieczeństwo typów. Poniżej znajdziesz podsumowanie kluczowych kroków integracji RTK z TypeScript:

  1. Definiowanie RootState i AppDispatch: Umożliwia silne typowanie stanu i funkcji dispatch w całej aplikacji.

  2. Tworzenie typowanych Hooków: Ułatwia używanie useDispatch i useSelector z poprawnym typowaniem.

  3. Definiowanie typów stanu i akcji w slice: Zapewnia, że każdy slice ma jasno określony typ stanu oraz typów akcji.

  4. Generowanie Typów akcji dla slice'ów: Umożliwia bezpieczne używanie akcji i ich typów.

  5. Praca z extraReducers: Zapewnia bezpieczną obsługę akcji generowanych przez thunk'y i inne slice’y.

Dzięki tym praktykom, możesz tworzyć skalowalne, łatwe w utrzymaniu i bezpieczne aplikacje z wykorzystaniem Redux Toolkit i TypeScript. Więcej informacji o integracji znajdziesz w dokumentacji Redux Toolkit.
