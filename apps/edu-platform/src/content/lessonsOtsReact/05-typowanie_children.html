W tej lekcji skupimy się na typowaniu propsa children w komponentach React.

Po ukończeniu tej lekcji będziesz umiał zastosować odpowiednie typy dla propsa children w React, dostosowane do specyficznych wymagań projektu.

Elastyczność React w przekazywaniu children

React oferuje dużą elastyczność w przekazywaniu children, co umożliwia tworzenie reużywalnych i elastycznych komponentów.

children mogą być pojedynczym elementem JSX, tablicą elementów, tekstem, a nawet funkcją. Takie różnorodne możliwości wymagają elastycznego podejścia do ich typowania w TypeScript.

Typowanie children za pomocą React.ReactNode

Najczęściej używanym typem do opisu children w komponentach jest React.ReactNode. Ten typ obejmuje większość możliwych wartości, które mogą być przekazane jako dzieci w React:

  • Typy proste (string, number, boolean)

  • Element React (React.ReactElement)

  • Tablice elementów React i zagnieżdżone struktury komponentów

  • null oraz undefined (umożliwiają renderowanie warunkowe)

import React from "react";

type CardProps = {
  children: React.ReactNode;
};

const Card = ({ children }): CardProps => (
  <div className="card">
    {children}
  </div>
);

W powyższym przykładzie komponent Card akceptuje dowolny typ children, który jest zgodny z React.ReactNode, co czyni go bardzo elastycznym.

Alternatywne metody typowania children

Chociaż React.ReactNode jest najczęściej używanym typem, czasami potrzebujemy bardziej precyzyjnego typowania.

ReactElement

Jednym z alternatywnych typów jest React.ReactElement, który akceptuje wyłącznie elementy Reacta lub HTML. Dzięki temu możemy ograniczyć children do konkretnego rodzaju elementów.

Typ ten akceptuje wyłącznie elementy Reacta lub HTML:

type CardProps = {
  children: React.ReactElement;
};

const Card= ({ children }: CardProps) => <div className="card">{children}</div>;

const CardContent = () => <p>Card content</p>;

const App = () => (
  <Card>
    <CardContent />
    {/* <span>Other content</span> */} {/* Błąd: zezwalamy na jeden komponent/element HTML */}
  </Card>
);

W tym przypadku komponent Card akceptuje tylko jeden element React jako children. Próba przekazania dodatkowego elementu, jak <span>, spowoduje błąd kompilacji.

Jeśli chcemy kontrolować liczbę przekazywanych elementów, możemy zdefiniować children jako tablicę elementów React:

type CardProps = {
  children: [React.ReactElement, React.ReactElement];
};

const Card = ({ children }: CardProps) => <div className="card">{children}</div>;

const App = () => (
  <Card>
    <CardContent />
    <span>Other content</span> {/* Teraz działa, ponieważ oczekujemy dokładnie dwóch elementów */}
  </Card>
);

Aby umożliwić dowolną liczbę elementów, można użyć typu React.ReactElement[]:

type CardProps = {
  children: React.ReactElement[];
};

JSX.Element

Ten typ to nakładka na omówione powyżej React.ReactElement, która jest dystrybuowana w samym TypeScript. Dla spójności z pozostałymi wariantami rekomendujemy stosowanie ReactElement.

Typowanie children jako funkcji

W niektórych przypadkach, szczególnie przy wzorcu #Render props, children mogą być funkcją zwracającą element Reacta. W takim scenariuszu typowanie wygląda nieco inaczej:

type CardProps = {
  children: (name: string) => React.ReactNode;
};

const Card = ({ children }: CardProps) => (
  <div className='card'>{children("Marcin")}</div>
);

const App = () => <Card>{(name) => <p>User: {name}</p>}</Card>;

Tutaj children jest funkcją, która przyjmuje parametr typu string i zwraca React.ReactNode, co pozwala na dynamiczne generowanie zawartości na podstawie przekazanych danych.

Przestarzałe typy children

Wraz z rozwojem Reacta i TypeScriptu, niektóre typy używane do typowania children zostały uznane za przestarzałe i niezalecane do użycia w nowych projektach. Przykładem są ReactText, ReactChild, ReactNodeArray oraz ReactFragment.

  • ReactText: Kiedyś używany dla children w postaci tekstu lub liczby, obecnie zaleca się bezpośrednie użycie string | number dla lepszej czytelności.

// Kiedyś
type CardProps = {
  children: React.ReactText;
}

// Teraz
type CardProps = {
  children: string | number;
};

  • ReactChild: Typ ten był używany jako połączenie ReactElement i tekstu, jednak teraz lepiej jest stosować React.ReactNode, który oferuje tę samą funkcjonalność w bardziej nowoczesny sposób.

// Kiedyś
type CardProps = {
  children: React.ReactChild;
}

// Teraz
type CardProps = {
  children: React.ReactNode;
};

  • ReactNodeArray i ReactFragment: ReactNodeArray zostało zastąpione przez React.ReactNode[], a ReactFragment przez Iterable<React.ReactNode>. W przypadku potrzeby użycia tych typów najlepiej jest korzystać z ich aktualnych odpowiedników.

Czy mogę ograniczyć children do wybranego komponentu?

Niestety nie.

Na poziomie wnioskowania o typach, każdy komponent jest finalnie postrzegany przez TypeScript jako JSX.Element (tj. ReactElement). Z tego powodu integracja TypeScript z React obecnie nie pozwala na ograniczenie children do wybranego komponentu.

Podsumowanie

Typowanie children w React za pomocą TypeScript daje nam możliwość kontrolowania, jakie wartości mogą być przekazywane do komponentów.

W zależności od potrzeb możemy wybrać bardziej elastyczne podejście z React.ReactNode, które pozwala na szeroki zakres typów, lub bardziej restrykcyjne, takie jak React.ReactElement, które umożliwiają dokładniejsze określenie oczekiwanych elementów.

Integracja TypeScript z React nie pozwala na ograniczenie children do wybranego komponentu.

Ważne jest także, aby unikać przestarzałych typów, które mogą prowadzić do niejasności i problemów w przyszłości.
