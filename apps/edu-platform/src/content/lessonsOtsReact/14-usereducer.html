useReducer

Hook useReducer to alternatywa dla useState, przydatna szczególnie w przypadkach, gdy logika zarządzania stanem jest bardziej skomplikowana lub wymaga odniesienia do poprzedniego stanu.

useReducer przyjmuje funkcję reducera (state, action) => newState, początkowy stan initialState, a opcjonalnie także funkcję inicjalizującą. Zwraca tablicę złożoną z bieżącego stanu oraz metody dispatch do aktualizacji stanu.

const [state, dispatch] = useReducer(reducerFunction, initialState, initializerFunction);

Kluczowe elementy useReducer to:

  1. Stan (state): Obiekt zawierający bieżący stan aplikacji.

  2. Akcja (action): Obiekt definiujący typ zmiany statu i ewentualne dane do modyfikacji stanu (payload).

  3. Dispatch: Funkcja, dzięki której dostarczamy akcje do reducera.

  4. Reducer: Funkcja aktualizująca stan na podstawie dostarczonej akcji.

  5. Funkcja inicjalizująca (opcjonalna): Funkcja, która inicjalizuje stan w oparciu o wartości zewnętrzne.

Więcej o działaniu reducerów dowiesz się z naszego szkolenia Opanuj Frontend: AI Edition.

Przykład typowania useReducer

TypeScript udostępnia wbudowany typ Reducer w pakiecie React, który ułatwia typowanie funkcji redukującej i zwiększa bezpieczeństwo typów. Możemy go zastosować dla przejrzystości i uniknięcia błędów.

Załóżmy, że mamy licznik, który można zwiększać lub zmniejszać o podaną wartość. Użyjemy do tego akcji increment i decrement, które będą wymagały payloadu określającego wartość zmiany.

  1. Definiowanie stanu: Stan jest definiowany jako obiekt reprezentujący wszystkie elementy wymagane w aplikacji:

interface CounterState {
  count: number;
}

  1. Definiowanie akcji: Akcje są rozróżniane przy pomocy unii typów, co umożliwia różne struktury dla różnych akcji:

type CounterAction =
  | { type: 'increment'; payload: number }
  | { type: 'decrement'; payload: number };

W tym przypadku increment i decrement są akcjami, które wymagają payload o typie number.

  1. Funkcja reducera: W pakiecie React znajdziesz wbudowany typ Reducer , który ułatwia typowanie reducera i zwiększa bezpieczeństwo typów. Możemy go zastosować dla przejrzystości i uniknięcia błędów:

import { Reducer } from 'react';

const counterReducer: Reducer<CounterState, CounterAction> = (state, action) => {
  switch (action.type) {
    case 'increment':
      return { ...state, count: state.count + action.payload };
    case 'decrement':
      return { ...state, count: state.count - action.payload };
    default:
      throw new Error(`Unhandled action type: ${action.type}`);
  }
};

  1. Implementacja komponentu: Komponent z wykorzystaniem useReducer i zdefiniowanych typów:

import React, { useReducer } from 'react';

const initialState: CounterState = { count: 0 };

const Counter = () => {
  const [state, dispatch] = useReducer(counterReducer, initialState);

  return (
    <div>
      <p>Count: {state.count}</p>
      <button onClick={() => dispatch({ type: 'increment', payload: 1 })}>+</button>
      <button onClick={() => dispatch({ type: 'decrement', payload: 1 })}>-</button>
    </div>
  );
};

export default Counter;

Obsługa akcji bez wymogu payload

W przypadku useReducer często występują akcje, które nie wymagają przekazywania dodatkowych danych (payload). Na przykład akcja reset może przywrócić stan do wartości początkowej bez konieczności użycia dodatkowych danych. Aby umożliwić różne struktury akcji, można zastosować discriminated unions, które pozwalają jasno określić, które akcje wymagają payload, a które nie.

Załóżmy, że rozwijamy licznik o funkcję resetowania. Akcja reset nie będzie wymagała payloadu.

  1. Definiowanie typu stanu i akcji:

    interface CounterState {
  count: number;
}

type CounterAction =
  | { type: 'increment'; payload: number }  // Wymaga payload
  | { type: 'decrement'; payload: number }  // Wymaga payload
  | { type: 'reset' };                      // Brak payload

    Dzięki zastosowaniu discriminated union w CounterAction każda akcja ma swoją określoną strukturę.

  2. Implementacja funkcji reducera: Reducer jest odpowiedzialny za przetwarzanie różnych typów akcji. Dzięki discriminated unions, TypeScript automatycznie rozpoznaje, które akcje wymagają payload, co minimalizuje ryzyko błędów.

    import { Reducer } from 'react';

const counterReducer: Reducer<CounterState, CounterAction> = (state, action) => {
  switch (action.type) {
    case 'increment':
      return { ...state, count: state.count + action.payload };
    case 'decrement':
      return { ...state, count: state.count - action.payload };
    case 'reset':
      return { count: 0 }; // Jeżeli skorzystamy tutaj z action.payload, kompilator zgłosi błąd: "Property 'payload' does not exist on type '{ type: "reset"; }'"
    default:
      throw new Error(`Unhandled action type: ${action.type}`);
  }
};

  3. Implementacja komponentu: W komponencie Counter możemy bezpiecznie wywoływać różne akcje z lub bez payload, ponieważ TypeScript wymusza poprawność typów dla każdej z nich. 

const initialState: CounterState = { count: 0 };

const Counter = () => {
  const [state, dispatch] = useReducer(counterReducer, initialState);

  return (
    <div>
      <p>Count: {state.count}</p>
      <button onClick={() => dispatch({ type: 'increment', payload: 1 })}>+</button>
      <button onClick={() => dispatch({ type: 'decrement', payload: 1 })}>-</button>
      <button onClick={() => dispatch({ type: 'reset' })}>Reset</button>
    </div>
  );
};
export default Counter;

Korzyści wynikające z tego podejścia:

  • Bezpieczeństwo typów: Dzięki discriminated unions, TypeScript jednoznacznie wymaga payload tylko wtedy, gdy jest on potrzebny, a pomija go, gdy akcja go nie potrzebuje.

  • Czytelność kodu: Każda akcja jest jasno określona, co ułatwia zrozumienie logiki reduktora.

  • Minimalizacja błędów: W przypadku dodania nowych akcji bez payload, TypeScript automatycznie zgłosi błąd, jeśli przypadkowo spróbujemy użyć payload tam, gdzie nie jest to wymagane.

Dzięki temu podejściu useReducer jest elastyczny i bezpieczny dla typów, umożliwiając tworzenie złożonych aplikacji React z bardziej szczegółowym kontrolowaniem struktury akcji.

Podsumowanie

W tej lekcji omówiliśmy wykorzystanie hooka useReducer jako alternatywy dla useState, szczególnie w kontekście zarządzania bardziej złożonym stanem aplikacji, który wymaga precyzyjnego śledzenia i modyfikacji. Omówiliśmy:

  1. Typowanie stanu i akcji: Nauczyliśmy się definiować struktury dla stanu (state) i akcji (action) z użyciem TypeScript, zapewniając jednoznaczność i bezpieczeństwo typów.

  2. Tworzenie reducera z typem Reducer: Poznaliśmy sposób użycia wbudowanego typu Reducer z paczki React, co ułatwia typowanie funkcji redukującej oraz minimalizuje ryzyko błędów wynikających z nieobsłużonych akcji.

  3. Obsługa akcji bez payloadu: Omówiliśmy, jak efektywnie typować akcje, które nie wymagają przekazywania danych (payload), takich jak akcja resetująca licznik, dzięki czemu reducer automatycznie rozpoznaje brakujące elementy tam, gdzie nie są one potrzebne. Dzięki użyciu unii dyskryminowanych każda akcja jest precyzyjnie określona, a wymagane elementy są jasno wskazane.

Dzięki tej lekcji masz solidną podstawę do korzystania z useReducer w aplikacjach React z TypeScriptem.
