W tej lekcji omówimy, jak efektywnie stosować TypeScript w projektach React, korzystając z hooków useMemo oraz useCallback. Zrozumienie tych hooków jest kluczowe dla optymalizacji wydajności aplikacji, zwłaszcza gdy pracujemy z kosztownymi obliczeniami lub przekazujemy funkcje do komponentów potomnych.

Hook useMemo

useMemo pozwala na memoizację (zapamiętywanie) wyniku kosztownych obliczeń. Polega to na tym, że funkcja przekaza do useMemo zostaje wykonana ponownie tylko wtedy, gdy zmienią się jej zależności. Jest to analogiczne do useEffect, który wykonuje efekt tylko wtedy, gdy zmieniają się określone zależności.

Rozważmy przykład, w którym chcemy obliczyć histogram kolorów obrazu. Obliczenie to jest kosztowne, dlatego chcemy je wykonać tylko wtedy, gdy dane obrazu (imageData) ulegną zmianie.

import React, { useMemo } from 'react';

interface ImageData {
  // Definicja właściwości ImageData
}

function getHistogram(image: ImageData): number[] {
  // Implementacja obliczania histogramu
  const histogram: number[] = [];
  // ... logika obliczania histogramu
  return histogram;
}

const HistogramComponent: { imageData: ImageData } = ({ imageData }) => {
  const histogram = useMemo<number[]>(() => getHistogram(imageData), [imageData]);

  return (
    <div>
      {histogram.map((count, index) => (
        <div key={index} style={{ height: count }}></div>
      ))}
    </div>
  );
};

  • Funkcja getHistogram przyjmuje ImageData i zwraca tablicę liczb reprezentujących histogram.

  • Hook useMemo memoizuje wynik getHistogram(imageData) i przechowuje go w histogram.

  • Obliczenie histogramu zostanie wykonane tylko wtedy, gdy imageData ulegnie zmianie.

Hook useCallback

useCallback to hook w React, który zwraca memoizowaną wersję funkcji. Jest to szczególnie przydatne, gdy przekazujemy funkcje do komponentów potomnych, ponieważ utrzymanie stabilną referencję funkcji między renderowaniami i zapobiega nadmiarowym rerenderom.

Przyjrzyjmy się przykładowi, w którym definiujemy funkcję, która wykonuje pewne działanie na podstawie przekazanych parametrów.

import React, { useCallback } from 'react';

const CallbackComponent = () => {
  const memoizedCallback = useCallback((param1: string, param2: number) => {
    console.log(param1, param2);
    return { ok: true };
  }, []);

  // Użycie memoizedCallback
  const handleClick = () => {
    const result = memoizedCallback('test', 42);
    console.log(result);
  };

  return <button onClick={handleClick}>Kliknij mnie</button>;
};

  • useCallback memoizuje funkcję, dzięki czemu jej referencja pozostaje stabilna między renderowaniami, o ile zależności się nie zmienią.

  • W przykładzie funkcja przyjmuje param1 typu string oraz param2 typu number i zwraca obiekt { ok: true }.

  • Dzięki TypeScript, typy parametrów i zwracany typ są jasno określone, co zwiększa bezpieczeństwo typów i ułatwia refaktoryzację.

Typowanie w useMemo i useCallback

useMemo

Kiedy korzystasz useMemo, warto upewnić się, że typ zwracany przez useMemo jest taki sam jak ten zwracany przez jego callbacka. React automatycznie inferuje typy, ale w razie potrzeby można je jawnie określić.

const memoizedValue = useMemo<number[]>(() => computeExpensiveValue(a, b), [a, b]);

useCallback

Podobnie jak useMemo, useCallback może być typowany jawnie lub polegać na inferencji TypeScript.

const memoizedCallback = useCallback(
  (param1: string, param2: number): { ok: boolean } => {
    console.log(param1, param2);
    return { ok: true };
  },
  [/* zależności */]
);

Różnie pomiędzy wersjami:

  • W React < 18, typowanie argumentów funkcji w useCallback domyślnie używa any[], co może prowadzić do błędów typów.

  • W React >= 18, typowanie zostało zmienione, aby być bardziej restrykcyjne, co wymusza jawne określenie typów argumentów funkcji.

Przykład błędu w React >= 18:

// @ts-expect-error Parameter 'e' implicitly has an 'any' type.
const handleEvent = useCallback((e) => {
  console.log(e);
}, []);

Poprawne typowanie:

const handleEvent = useCallback((e: React.MouseEvent<HTMLButtonElement>) => {
  console.log(e);
}, []);

Podsumowanie

W tej lekcji omówiliśmy, jak używać hooków useMemo oraz useCallback w React z TypeScript, aby zoptymalizować wydajność aplikacji:

  • useMemo służy do memoizacji wyników kosztownych obliczeń, wykonując je tylko wtedy, gdy zmieniają się zależności.

  • useCallback pozwala na memoizację funkcji, utrzymując ich referencje stabilne między renderowaniami, co jest szczególnie przydatne przy przekazywaniu funkcji do komponentów potomnych.

  • Typowanie tych hooków z TypeScript jest proste dzięki typom wbudowanym i inferencji kompilatora.

  • Różnice pomiędzy wersjami React wpływają na sposób typowania funkcji w useCallback, co warto mieć na uwadze przy migracji projektu do wersji >= 18.
