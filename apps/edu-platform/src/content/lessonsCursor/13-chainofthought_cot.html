Chain-of-Thought (CoT) to technika zaprojektowana w celu poprawy zdolności rozumowania dużych modeli językowych poprzez promptowanie ich do generowania procesu rozumowania krok po kroku przed uzyskaniem ostatecznej odpowiedzi.

Technika ta ma na celu naśladowanie sposobu, w jaki ludzie rozwiązują problemy, dzieląc złożone zadania na mniejsze, łatwiejsze do wykonania kroki.

Jak używać?

Pomyśl o CoT jako o prowadzeniu LLM do „pokazania swojej pracy” zamiast po prostu dostarczania ostatecznej odpowiedzi. Jest to szczególnie cenne w inżynierii oprogramowania, gdzie zrozumienie logiki stojącej za rozwiązaniem jest równie ważne jak samo rozwiązanie.

Oto jak CoT działa w praktyce:

  • Prompt: Kiedy stawiasz problem przed LLM, powinieneś dołączyć frazę lub instrukcję, która zachęci model do myślenia krok po kroku. Typowe przykłady takich zwrotów to: „Zastanówmy się krok po kroku”, „Rozważmy to krok po kroku, aby upewnić się, że mamy właściwą odpowiedź” lub „Najpierw zastanówmy się nad tym logicznie”.

  • Proces rozumowania: W odpowiedzi na prompt, LLM wygeneruje serię pośrednich kroków rozumowania, które prowadzą do ostatecznej odpowiedzi. Kroki te zapewniają wgląd w proces myślowy modelu, ułatwiając zrozumienie, w jaki sposób doszedł on do konkretnego rozwiązania.●

  • Zero-Shot CoT: Najprostsza forma CoT, Zero-Shot CoT, polega na dołączeniu do prompta frazy skłaniającej do myślenia bez podawania przykładów. Choć prosta w implementacji, może nie zawsze prowadzić do najdokładniejszych wyników, szczególnie w przypadku złożonych zadań.

  • Few-Shot CoT: Aby jeszcze bardziej zwiększyć skuteczność CoT, można użyć Few-Shot CoT, który polega na dostarczeniu LLM kilku przykładów pytań w połączeniu z odpowiadającym im rozumowaniem łańcuchowym i odpowiedziami. Pozwala to modelowi lepiej zrozumieć pożądany proces rozumowania i potencjalnie wygenerować bardziej dokładne i logiczne wyniki.

Rozważmy scenariusz, w którym budujesz witrynę e-commerce i musisz zaimplementować algorytm obliczania rabatów. Możesz użyć promptowania CoT, aby pomóc LLM wygenerować bardziej solidne i zrozumiałe rozwiązanie.

Podstawowy prompt: 

Napisz funkcję obliczającą ostateczną cenę produktu po zastosowaniu rabatu w języku JavaScript.

Odpowiedź modelu:

Rozumiem, że chcesz funkcję w JavaScript, która oblicza końcową cenę produktu po uwzględnieniu rabatu. Oto implementacja takiej funkcji:

function calculateFinalPrice(originalPrice, discountPercentage) {
  if (originalPrice < 0 || discountPercentage < 0 || discountPercentage > 100) {
    return "Invalid input";
  }
  
  const discountAmount = originalPrice * (discountPercentage / 100);
  const finalPrice = originalPrice - discountAmount;
  
  return finalPrice.toFixed(2);
}

Ta funkcja:
1. Przyjmuje dwa argumenty: originalPrice (początkowa cena produktu) i discountPercentage (wartość rabatu w procentach).
2. Sprawdza, czy dane wejściowe są prawidłowe (cena nie może być ujemna, rabat musi być między 0 a 100%).
3. Oblicza kwotę rabatu.
4. Odejmuje kwotę rabatu od ceny wyjściowej.
5. Zwraca cenę końcową, zaokrągloną do dwóch miejsc po przecinku.

Czy chciałbyś, żebym wyjaśnił kod bardziej szczegółowo lub pokazał przykład użycia tej funkcji?

CoT Prompt:

Pomyślmy krok po kroku. Napisz funkcję obliczającą ostateczną cenę produktu po zastosowaniu rabatu w języku JavaScript. Najpierw sprawdź, czy rabat jest wartością procentową czy stałą. Następnie...

Odpowiedź modelu:

Rozumiem. Napiszę funkcję w JavaScript, która oblicza ostateczną cenę produktu po zastosowaniu rabatu. Przejdźmy przez to krok po kroku:

function calculateDiscountedPrice(originalPrice, discount) {
  // Sprawdzenie, czy rabat jest wartością procentową
  if (typeof discount === 'string' && discount.endsWith('%')) {
    // Konwersja rabatu procentowego na liczbę
    const discountPercentage = parseFloat(discount) / 100;
    
    // Obliczenie ceny po rabacie procentowym
    return originalPrice - (originalPrice * discountPercentage);
  } else {
    // Traktowanie rabatu jako wartości stałej
    const discountAmount = parseFloat(discount);
    
    // Sprawdzenie, czy rabat nie jest większy niż cena oryginalna
    if (discountAmount > originalPrice) {
      return 0; // Cena nie może być ujemna
    }
    
    // Obliczenie ceny po rabacie stałym
    return originalPrice - discountAmount;
  }
}

// Przykłady użycia:
console.log(calculateDiscountedPrice(100, '20%')); // Rabat procentowy
console.log(calculateDiscountedPrice(100, 30));    // Rabat stały

Oto wyjaśnienie krok po kroku:
1. Definiujemy funkcję calculateDiscountedPrice, która przyjmuje dwa parametry: originalPrice (oryginalna cena) i discount (rabat).
2. Sprawdzamy, czy rabat jest wartością procentową. Robimy to, sprawdzając, czy discount jest stringiem kończącym się znakiem '%'.
3. Jeśli rabat jest procentowy:
- Konwertujemy go na liczbę dziesiętną (np. 20% na 0.2).
- Obliczamy cenę po rabacie, odejmując od ceny oryginalnej iloczyn ceny i rabatu procentowego.
4. Jeśli rabat nie jest procentowy, traktujemy go jako wartość stałą:
- Konwertujemy rabat na liczbę.
- Sprawdzamy, czy rabat nie jest większy niż cena oryginalna. Jeśli jest, zwracamy 0 (cena nie może być ujemna).
- Obliczamy cenę po rabacie, odejmując wartość rabatu od ceny oryginalnej.
5. Na końcu dodałem przykłady użycia funkcji dla obu typów rabatów.

Czy chciałbyś, żebym coś wyjaśnił lub rozwinął w tym kodzie?

Używając promptu CoT, zachęcasz LLM do podzielenia obliczeń rabatu na mniejsze kroki, dzięki czemu wynikowy kod jest bardziej przejrzysty i łatwiejszy do zrozumienia. Dzięki temu lepiej zrozumiesz kod wygenerowany przez LLM i będziesz mógł wziąć za niego odpowiedzialność. Bardzo możliwe, że kod będzie również wyższej jakości: wzrośnie jego czytelność i spadnie ryzyko błędów.

Przykładowe zastosowania

Jako web developerzy możemy wykorzystać technikę Chain of Thought w różnych aspektach naszej pracy, szczególnie w zadaniach wymagających złożonego rozumowania i rozwiązywania problemów:

  • Debugowanie: Pomaga w systematycznym podejściu do identyfikacji i rozwiązywania błędów w kodzie.

  • Optymalizacja wydajności: Wspiera proces analizy i poprawy wydajności aplikacji poprzez rozbicie problemu na mniejsze, bardziej zarządzalne kroki.

  • Projektowanie architektury: Umożliwia bardziej strukturyzowane podejście do projektowania złożonych systemów i ich komponentów.

  • Refaktoryzacja kodu: Wspomaga proces reorganizacji istniejącego kodu poprzez systematyczne rozważanie każdego kroku i jego konsekwencji.

Zrozumienie i stosowanie CoT pozwala nam na bardziej efektywne wykorzystanie AI w procesie tworzenia oprogramowania, szczególnie w przypadku złożonych problemów.

🥳 Korzyści

  • Lepsze zrozumienie procesu myślowego: Pozwala developerom lepiej zrozumieć, jak AI dochodzi do konkretnych rozwiązań.

  • Wyższa jakość kodu: Prowadzi do generowania bardziej przemyślanych i lepiej ustrukturyzowanych rozwiązań.

  • Łatwiejsze debugowanie: Ułatwia identyfikację potencjalnych błędów w logice poprzez analizę każdego kroku rozumowania.

  • Edukacyjny aspekt: Może służyć jako narzędzie do nauki, pokazując krok po kroku, jak rozwiązywać złożone problemy programistyczne.

🫣 Wyzwania

  • Zwiększona złożoność promptów: Wymaga tworzenia bardziej rozbudowanych i precyzyjnych promptów.

  • Dłuższy czas generowania odpowiedzi: Proces może być bardziej czasochłonny ze względu na generowanie szczegółowego procesu myślowego.

  • Potencjalne przeciążenie informacjami: Istnieje ryzyko generowania zbyt szczegółowych wyjaśnień, które mogą być przytłaczające dla niektórych użytkowników.

🧠 Kluczowe wnioski

  1. Selektywne stosowanie: CoT jest najbardziej efektywne w złożonych scenariuszach, gdzie proces rozumowania jest równie ważny jak końcowy rezultat.

  2. Balans między szczegółowością a zwięzłością: Ważne jest znalezienie odpowiedniej równowagi między dostarczaniem szczegółowego procesu myślowego a utrzymaniem czytelności i użyteczności wygenerowanego kodu.

  3. Weryfikacja wyników: Mimo że CoT może prowadzić do bardziej przemyślanych rozwiązań, zawsze należy krytycznie oceniać i testować wygenerowany kod.

  4. Narzędzie wspomagające, nie zastępujące: CoT powinno być traktowane jako wsparcie dla procesu myślowego developera, a nie jako całkowite zastąpienie ludzkiego rozumowania i doświadczenia.

🧑🏻‍💻 Ćwiczenie

Twoim zadaniem jest napisanie funkcji JavaScript, która waliduje i formatuje numer telefonu. Zamiast prosić AI o gotowe rozwiązanie, użyjesz techniki CoT, aby uzyskać szczegółowe wyjaśnienie procesu myślowego prowadzącego do rozwiązania.

Kroki

  1. Sformułuj prompt CoT: Napisz prompt, który zachęci model AI do użycia techniki Chain of Thought przy tworzeniu funkcji walidującej i formatującej numer telefonu. Twój prompt powinien zawierać:

    • Opis zadania

    • Prośbę o myślenie krok po kroku

    • Sugestię rozważenia różnych aspektów problemu (np. różne formaty numerów, obsługa błędów)

  2. Analiza odpowiedzi: Po otrzymaniu odpowiedzi od AI, przeanalizuj ją pod kątem:

    • Jasności procesu myślowego

    • Kompletności rozwiązania

    • Potencjalnych luk lub nieścisłości

  3. Iteracja: Jeśli odpowiedź nie jest satysfakcjonująca, zmodyfikuj swój prompt i spróbuj ponownie. Skup się na aspektach, które wymagają dopracowania.

  4. Implementacja: Na podstawie otrzymanego wyjaśnienia, zaimplementuj funkcję samodzielnie. Porównaj swoją implementację z proponowanym rozwiązaniem.

  5. Refleksja: Zastanów się nad procesem i odpowiedz na następujące pytania:

    • Jak technika CoT wpłynęła na jakość otrzymanego rozwiązania?

    • Jakie wyzwania napotkałeś podczas formułowania promptu CoT?

    • Jak możesz wykorzystać tę technikę w swojej codziennej pracy jako web developer?

Wskazówki

  • Zachęć AI do rozważenia różnych scenariuszy i przypadków brzegowych.

  • Poproś o wyjaśnienie każdej decyzji projektowej.

  • Zwróć uwagę na to, jak AI radzi sobie z potencjalnymi wyzwaniami (np. różne formaty międzynarodowe).

Kryteria sukcesu

  • Twój prompt skutecznie zachęca AI do użycia techniki CoT.

  • Otrzymujesz szczegółowe, krok po kroku, wyjaśnienie procesu tworzenia funkcji.

  • Jesteś w stanie zaimplementować funkcję na podstawie otrzymanego wyjaśnienia.
