Wprowadzenie

In-context learning to kluczowa koncepcja w prompt engineering, która umożliwia modelom językowym szybkie dostosowanie się do nowych zadań bez konieczności ponownego treningu. Ta zdolność opiera się na fundamentalnej właściwości LLM-ów: ich umiejętności uczenia się i adaptacji na podstawie kontekstu dostarczanego w promptach.

Znaczenie przykładów dla LLM-ów

LLM-y, takie jak GPT, są trenowane na ogromnych ilościach tekstu, co daje im szeroką wiedzę ogólną. Jednak ta wiedza jest "uśredniona" i nie zawsze bezpośrednio aplikowalna do konkretnych, specjalistycznych zadań. Przykłady w promptach pełnią kluczową rolę:

  1. Ukierunkowanie: Przykłady pomagają "ukierunkować" ogólną wiedzę modelu na konkretne zadanie.

  2. Demonstracja: Pokazują modelowi jakiego rodzaju wynik jest oczekiwany.

  3. Kontekst: Dostarczają kontekstu, który może być kluczowy dla prawidłowego wykonania zadania.

  4. Format: Ilustrują pożądany format lub strukturę odpowiedzi.

Zrozumienie roli przykładów pozwala nam efektywniej wykorzystywać różne techniki promptingu, takie jak zero-shot i few-shot, które omówimy poniżej.

Zero-shot Prompting

Zero-shot prompting polega na dostarczeniu LLM instrukcji zadania bez żadnych konkretnych przykładów. Model opiera się wyłącznie na swojej wstępnie wytrenowanej wiedzy, aby wygenerować pożądany wynik.

Jak używać?

Załóżmy, że chcemy stworzyć funkcję JavaScript do walidacji adresu email. Oto jak mógłby wyglądać zero-shot prompt:

Wygeneruj funkcję JavaScript do sprawdzania poprawności adresów e-mail.

🥳 Korzyści

  • Szybkość i wydajność: Jest to najszybsza metoda, ponieważ nie wymaga tworzenia ani dobierania przykładów.

  • Przydatność w zadaniach eksploracyjnych: Doskonale nadaje się do szybkiego zbadania możliwości LLM lub uzyskania ogólnego wrażenia na temat jego potencjalnej wydajności dla danego zadania.

🫣 Wyzwania

  • Nieprzewidywalna dokładność: Poleganie wyłącznie na wstępnie wytrenowanej wiedzy LLM może prowadzić do mniej dokładnych lub trafnych wyników, szczególnie w przypadku bardzo specyficznych lub złożonych zadań.

Few-shot Prompting

Few-shot prompting polega na dostarczeniu LLM instrukcji zadania wraz z kilkoma przykładami ilustrującymi pożądane mapowanie wejścia-wyjścia. Pomaga to modelowi lepiej zrozumieć zadanie i wygenerować dokładniejsze i trafniejsze wyniki.

Jak używać?

Kontynuując przykład z walidacją adresu email, oto jak mógłby wyglądać few-shot prompt:

Poniżej znajdują się dwie funkcje JavaScript do walidacji adresów email. Wygeneruj nową funkcję, która łączy ich funkcjonalności i dodaje sprawdzenie, czy domena ma co najmniej jedną kropkę.

Funkcja 1:
function validateEmail(email) {
  return /^[^\s@]+@[^\s@]+$/.test(email);
}

Funkcja 2:
function checkEmail(email) {
  const atIndex = email.indexOf('@');
  return atIndex > 0 && atIndex < email.length - 1;
}

🥳 Korzyści

  • Większa dokładność i trafność: Dostarczanie przykładów pomaga LLM w lepszym zrozumieniu zadania, prowadząc do bardziej wiarygodnych i kontekstowo odpowiednich wyników.

  • Szczegółowa kontrola: Pozwala na bardziej precyzyjną kontrolę nad wynikami pracy LLM poprzez pokazanie konkretnych wzorców lub struktur, które mają być przestrzegane.

🫣 Wyzwania

  • Kluczowe znaczenie wyboru i kolejności przykładów: Wybór i prezentacja przykładów ma znaczący wpływ na wydajność. Należy dokładnie rozważyć wybór przykładów, które są reprezentatywne, różnorodne i logicznie uporządkowane.

  • Zwiększony narzut na promptowanie: Tworzenie skutecznych promptów z przykładami wymaga więcej czasu i wysiłku w porównaniu do promptowania bez przykładów.

—

In-context learning jest kluczową koncepcją w prompt engineeringu, umożliwiającą modelom językowym szybką adaptację do nowych zadań bez konieczności ponownego treningu. Technika ta opiera się na zdolności LLM-ów do uczenia się z kontekstu dostarczanego w promptach, co pozwala na efektywne wykorzystanie ich ogólnej wiedzy w specyficznych zastosowaniach.

🧠 Kluczowe wnioski

  1. Przykłady w promptach: pełnią istotną rolę w ukierunkowaniu wiedzy modelu, demonstracji oczekiwanych wyników, dostarczaniu kontekstu i ilustracji pożądanego formatu odpowiedzi.

  2. Zero-shot prompting: polega na dostarczeniu instrukcji bez przykładów, oferując szybkość i przydatność w zadaniach eksploracyjnych, ale może prowadzić do mniej przewidywalnych wyników.

  3. Few-shot prompting: wykorzystuje kilka przykładów do ilustracji zadania, zapewniając większą dokładność i kontrolę nad wynikami, ale wymaga starannego doboru przykładów.

  4. Wybór między zero-shot a few-shot: zależy od złożoności zadania, wymagań dotyczących dokładności oraz kompromisu między szybkością a precyzją.

  5. Skuteczne wykorzystanie in-context learning: wymaga eksperymentowania, oceny wyników i iteracyjnego udoskonalania technik promptingu.

Kroki

  1. Zero-shot prompting

Poproś LLM o wygenerowanie komponentu React, który wyświetla wykres słupkowy pokazujący liczbę odwiedzin strony w ciągu ostatnich 7 dni. Komponent powinien przyjmować dane jako prop i używać biblioteki recharts do renderowania wykresu.

  1. Few-shot prompting

Dostarcz LLM następujący przykład komponentu React:

import React from 'react';
import { PieChart, Pie, Cell, ResponsiveContainer, Legend } from 'recharts';

const COLORS = ['#0088FE', '#00C49F', '#FFBB28', '#FF8042'];

const DeviceUsageChart = ({ data }) => {
  return (
    <div style={{ width: '100%', height: 300 }}>
      <ResponsiveContainer>
        <PieChart>
          <Pie
            data={data}
            cx="50%"
            cy="50%"
            labelLine={false}
            outerRadius={80}
            fill="#8884d8"
            dataKey="value"
          >
            {data.map((entry, index) => (
              <Cell key={`cell-${index}`} fill={COLORS[index % COLORS.length]} />
            ))}
          </Pie>
          <Legend />
        </PieChart>
      </ResponsiveContainer>
    </div>
  );
};

export default DeviceUsageChart;

Następnie poproś LLM o wygenerowanie nowego komponentu React, który wyświetla wykres liniowy pokazujący trendy w czasie ładowania strony w ciągu ostatniego miesiąca. Komponent powinien również używać recharts i przyjmować dane jako prop.
