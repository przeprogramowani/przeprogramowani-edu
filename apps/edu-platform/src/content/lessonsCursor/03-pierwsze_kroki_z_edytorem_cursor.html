Wstęp

W poprzedniej lekcji poznałeś teoretyczne założenia generatywnego AI, które z punktu widzenia programisty będą wpływać na rezultaty pracy z modelami. Dwa najważniejsze cele, które zrealizujemy w tej lekcji, to:

Ta lekcja pozwoli nam skonfigurować środowisko pracy i przygotować się do scenariuszy praktycznych, które przygotowaliśmy w dalszej części szkolenia.

Instalacja edytora Cursor

Zacznijmy od instalacji edytora i założenia darmowego konta - aby to zrobić, zrealizuj następujące kroki:

  1. Wejdź na stronę https://www.cursor.com/

  2. Uruchom pobieranie edytora klikając w przycisk “Download” / “Download for Free”

  3. Załóż darmowe konto korzystając z formularza na stronie https://authenticator.cursor.sh/sign-up

Założenie konta to krok, który wg twórców Cursora pomaga im zapewnić więcej bezpieczeństwa i kontroli nad zapytaniami do API. Konto jest w pełni darmowe, a proces nie wymaga konfigurowania płatności lub podawania danych do karty płatniczej.

Całą komunikację z AI zrealizujemy w oparciu o własne klucze do API, które pozwolą na komunikację z modelami firm takich jak OpenAI (modele GPT-4o mini oraz GPT-4o) oraz Anthropic (Claude 3.5 Sonnet).

Komunikacja z API

Ważne, żebyś rozumiał jak działa komunikacja z modelami AI realizowana w edytorze Cursor.

Ma ona trzy etapy:

  1. Twoje zapytanie i kontekst - wysyłane z poziomu edytora

  2. Ulepszenia i oznaczenia prompta - logika realizowana przez backend Cursora

  3. Generowanie odpowiedzi z modelu - realizowane przez usługi OpenAI / Anthropic / etc.

Cała logika biznesowa i “magia Cursora” jest realizowana przez środkowy element powyższego schematu. Prompty, które wysyłasz z poziomu edytora, są odpowiednio modyfikowane i optymalizowane przez backendowe serwisy Cursora, aby platformy i modele AI (ostatnia część procesu) mogły generować dla ciebie odpowiedzi najwyższej jakości.

Prywatność

Powyższy schemat pozwala też zrozumieć, dlaczego prywatność w Cursorze trzeba analizować z dwóch perspektyw - backendu edytora oraz docelowych platform AI.

Tryb prywatny, który włączyliśmy na etapie instalacji (lub poprzez Cursor Settings → General → Privacy Mode) wpływa na politykę backendu i serwerów Cursora, ale nie wpływa na politykę API OpenAI czy Anthropica.

Na dzisiaj, najpopularniejsze API działają zgodnie z następującymi regułami prywatności:

  • OpenAI API - twoje dane i prompty nie są wykorzystywane w procesie treningu i ulepszania modeli, ale same dane mogą pozostawać na serwerach do 30 dni.

  • Anthropic API - twoje dane i prompty nie są wykorzystywane w procesie treningu i ulepszania modeli, ale same dane mogą pozostawać na serwerach do 30 dni (lub do 2 lat, jeśli prompt zostanie oznaczony jako naruszający zasady bezpieczeństwa).

Rekomendujemy włączenie trybu prywatnego, a dla firm - rozważenie wejścia na plan biznesowy, który dodatkowo wymusza tzw. “0-data retention” na OpenAI i Anthropic API.

Więcej o prywatności znajdziesz na poniższych stronach:

  • https://www.cursor.com/security

  • https://www.cursor.com/privacy

Generowanie klucza do OpenAI API (modele GPT-4o, GPT-4o mini oraz o1)

W przypadku firmy OpenAI i modeli takich jak GPT-4o, cały proces wygląda bardzo podobnie. Jeśli chcesz, możesz przejść poniższe kroki zgodnie z instrukcją lub pominąć je na rzecz modelu Claude i wrócić tutaj nieco później.

Zaczynamy od założenia konta na stronie https://platform.openai.com/. Następnym krokiem jest skonfigurowanie metod płatności i doładowanie twojego konta. Sugerowana kolejność konfiguracji płatności to odpowiednio:

  • Dane adresowe i dane do faktury - w zakładce Billing → Preferences

  • Metody płatności - w zakładce Billing → Payment Methods

  • Doładowanie i stan konta - w zakładce Billing → Overview (przycisk Add to credit balance)

Mając doładowane konto, możemy przejść do wygenerowania klucza do API.

Podobnie jak w przypadku Anthropic API, stworzymy dedykowaną konfigurację pod Cursora - wykorzystamy tzw. Projekty (Projects), które pozwolą nam bardziej precyzyjne śledzić zużycie i aktywność zapytań z poszczególnymi kluczami.

Korzystając z menu w górnej sekcji, tworzymy dedykowany projekt (przycisk “Create project”):

Konfiguracja reguł i oczekiwań względem AI

Zgodnie z najnowszymi rekomendacjami (Cursor 0.45+), instrukcje dla AI powinno się umieszczać w repozytorium projektu, w folderze .cursor/rules, a każda powinna dotyczyć jednego aspektu współpracy z AI, np.:

  • bootstrapowania projektu

  • tworzenia testów

  • tworzenia komponentów client-side

  • tworzenia endpointów na backendzie

Cały proces jest mocno inspirowany Single Responsibility Principle - jeśli w twoim projekcie powtarzasz tworzenie lub modyfikację określonego typu pliku, struktury lub funkcji, utwórz regułę precyzującą własne oczekiwania.

Dodawanie reguł

Każdą z reguł możesz skonfigurować z poziomu konsoli Cursora, lub w folderze .cursor/rules (pliki w formacie .mdc):

Po wybraniu tej opcji zostaniesz przeniesiony do kreatora reguł, które po utworzeniu będą dodawane do nowych zapytań.

W ustawieniach Cursora, w sekcji General → Rules for AI znajdziesz podsumowanie wszystkich utworzonych reguł. Edytor będzie wybierał te najbardziej pasujące do konkretnych poleceń, starając się rozszerzyć kontekst i wpłynąć na otrzymanie rezultatów wysokiej jakości:

Przykładowa reguła dotycząca tworzenia endpointów w Astro może dotyczyć:

  • Wykorzystywania TypeScriptu w preferowany sposób

  • Używania jawnych typów oraz unikania typu any

  • Bazowania na wybranym fragmencie kodu

  • Dodatkowej dokumentacji zawierającej informacje wychodzącej poza to, co potrafi sam model AI

// .cursor/rules/astro-endpoint.mdc

# Your rule content

- Create valid Astro endpoints based on following example
- Keep in mind that HTTP Methods need to be uppercased
- Always use TypeScript
- Make sure types are sound and valid (avoid any as much as possible)

```typescript
import type { APIRoute } from "astro";

export const POST: APIRoute = async ({ request }) => {
  const data = await request.formData();
  const name = data.get("name");
  const email = data.get("email");
  const message = data.get("message");
  // Validate the data - you'll probably want to do more than this
  if (!name || !email || !message) {
    return new Response(
      JSON.stringify({
        message: "Missing required fields",
      }),
      { status: 400 }
    );
  }
  // Do something with the data, then return a success response
  return new Response(
    JSON.stringify({
      message: "Success!"
    }),
    { status: 200 }
  );
};
```

Dzięki dedykowanym instrukcjom zredukujesz efekt halucynacji i podniesiesz skuteczność odpowiedzi, jakie generuje dla ciebie Cursor. Tutaj liczy się rownież praktyka i obserwacja zachowań modelu - zawsze, kiedy zauważysz odstępstwo od odpowiedzi wysokiej jakości, możesz sprecyzować swoje oczekiwania dodając nową instrukcję (nauka modelu poprzez praktykę).

Autocomplete

Cursor oferuje autorskie rozwiązanie do autouzupełniania kodu o nazwie Cursor Tab, które jest dostępne wyłącznie w płatnym planie Pro. Szczegółowo przedstawimy je w lekcji IV.

Jeśli chcesz korzystać z Cursora w oparciu o własny klucz API, musisz wybrać autocomplete w formie wtyczki. Najpopularniejsze opcje to:

  • Github Copilot

  • Codeium

  • Supermaven

Po przeanalizowaniu tych rozwiązań, naszą rekomendacją jest darmowy Supermaven. Oto porównanie kluczowych aspektów:

  1. Szybkość: Supermaven wyróżnia się wyjątkowo szybkim działaniem. Użytkownicy zauważają, że nie doświadczają typowych opóźnień znanych z innych narzędzi.

  2. Jakość sugestii: Supermaven oferuje wysoką trafność podpowiedzi, często generując poprawny kod bez konieczności modyfikacji.

  3. Okno kontekstowe: Supermaven posiada imponujące okno kontekstowe o wielkości 300 000 tokenów w darmowym planie, a nawet 1 miliona tokenów w wersji Pro. To znacznie więcej niż konkurencja, co przekłada się na lepsze zrozumienie kontekstu projektu.

  4. Wszechstronność: Supermaven radzi sobie dobrze z różnymi językami programowania i zadaniami, od prostego autouzupełniania po generowanie całych funkcji.

Biorąc pod uwagę powyższe zalety, zwłaszcza szybkość działania, wysoką jakość sugestii oraz duże okno kontekstowe, rekomendujemy Supermaven jako najlepszą opcję autocomplete dla użytkowników Cursor korzystających z własnego klucza API.

Aby zainstalować go w edytorze wejdź w Extensions/Rozszerzenia i wyszukaj plugin: Supermaven.
