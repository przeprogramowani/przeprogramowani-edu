# QA Mode Debug Overlay — Implementation Plan

## Overview

Add a dedicated QA/debug mode to 10x Explorers, activated by loading the game with `?qa` in the URL. When active, a floating semi-transparent panel appears in the top-left corner showing live game state (flags, quests, XP, position, map) and provides a "Reset State" action button. Works in all builds (dev + production).

## Current State Analysis

- No URL parameter parsing exists anywhere in the game code
- No debug overlay or QA tools exist — only `import.meta.env.DEV` for physics debug and console.log statements
- Game state lives in `game.registry` and syncs via `STATE_CHANGED` events on the Phaser event bus
- `PhaserGame.svelte` is the top-level Svelte component that creates the Phaser instance and renders `SmartTerminal`
- State persistence uses `localStorage` key `10x-explorers-demo-state`
- `createDefaultState()` in `GameStateManager.ts` already provides a clean starting state

### Key Discoveries:

- `src/explorers/PhaserGame.svelte` — Svelte bridge, creates game, handles global keyboard events, renders SmartTerminal overlay
- `src/explorers/state/GameStateManager.ts` — `loadState()`, `saveState()`, `createDefaultState()`
- `src/explorers/config/constants.ts:37` — `SAVE_KEY = '10x-explorers-demo-state'`
- `src/explorers/events/GameEvents.ts` — `STATE_CHANGED` event with `StateChangedPayload`
- `src/explorers/state/types.ts` — `DemoGameState` interface

## Desired End State

When a developer navigates to `/explorers?qa`:

1. A floating debug panel appears in the top-left corner (semi-transparent, dark, monospace)
2. The panel shows a **live state inspector** updating in real-time:
   - Current map + position (x, y)
   - Flags (listed as tags/chips)
   - Quest state (active quest ID, completed quest IDs)
   - XP value
   - Terminal unlocked (yes/no)
   - Intro seen (yes/no)
   - Map unlocked (yes/no)
3. A **"Reset State"** button clears localStorage and reloads the page
4. Without `?qa`, the game behaves exactly as before — zero impact on normal gameplay

### Verification:

- Navigate to `/explorers?qa` → debug panel visible in top-left
- Navigate to `/explorers` → no debug panel
- Play the game with QA panel open → state updates live as you move, interact, complete quests
- Click "Reset State" → localStorage cleared, page reloads to fresh start
- Build for production (`npm run build`) → QA mode still works with `?qa` param

## What We're NOT Doing

- **No state presets** — only a full reset to default state (user confirmed minimal approach)
- **No state editing** — the inspector is read-only; state manipulation is via reset only
- **No keyboard shortcut to toggle** — panel is always visible when `?qa` is in URL
- **No Phaser-side debug scene** — overlay is pure Svelte DOM, not a Phaser scene
- **No map teleportation** — developers use door transitions normally
- **No flag toggling UI** — use browser devtools or terminal commands for granular manipulation

## Implementation Approach

Single-phase implementation. The QA overlay is a standalone Svelte component rendered conditionally in `PhaserGame.svelte` based on URL parameter detection. It subscribes to Phaser's event bus for live state updates.

## Critical Implementation Details

### Timing & Lifecycle Considerations

- **QA param detection**: Read `window.location.search` once in `PhaserGame.svelte`'s `onMount`, before game creation. Store as a reactive variable.
- **State subscription**: The QA overlay subscribes to `game.events` for `STATE_CHANGED` after `game` is initialized. Must clean up listener in `onDestroy`.
- **Initial state**: On mount, read initial state from `game.registry.get('demoGameState')`. Subsequent updates arrive via events.

### User Experience Specification

- **Panel position**: Fixed top-left, 16px margin from edges
- **Panel size**: ~320px wide, auto-height based on content
- **Visual style**: `bg-gray-950/85` (dark, 85% opacity), `border border-gray-700`, rounded corners, monospace font (`font-mono`), small text (`text-xs`)
- **State display**: Compact key-value pairs. Flags as inline comma-separated list. Quests as "Active: X | Done: [a, b]"
- **Reset button**: Red accent, bottom of panel. Click → `window.confirm()` → clear localStorage → `window.location.reload()`
- **Z-index**: Above game canvas and terminal overlay (z-50 or higher)
- **Non-interactive with game**: Panel uses `pointer-events-auto` but doesn't interfere with Phaser input (it's outside the canvas)

### State Management Sequencing

- **N/A**: The QA overlay is read-only. It passively observes state changes via the existing event bus. No state mutations except the reset action.

### Debug & Observability Plan

- **N/A**: This IS the debug/observability tool. No additional instrumentation needed.

## Phase 1: QA Mode Overlay

### Overview

Create a `QaOverlay.svelte` component and wire it into `PhaserGame.svelte` conditionally based on `?qa` URL parameter.

### Changes Required:

#### 1. QA Overlay Component

**File**: `src/explorers/QaOverlay.svelte`

New Svelte component that receives the Phaser `game` instance as a prop and renders a floating debug panel.

```svelte
<script lang="ts">
  import { onMount, onDestroy } from 'svelte';
  import { GameEvents } from './events/GameEvents';
  import { SAVE_KEY } from './config/constants';
  import type { DemoGameState } from './state/types';

  export let game: Phaser.Game;

  let state: DemoGameState | null = null;

  function onStateChanged(payload: { state: DemoGameState }) {
    state = payload.state;
  }

  function resetState() {
    if (window.confirm('Reset all game state? This will clear localStorage and reload.')) {
      localStorage.removeItem(SAVE_KEY);
      window.location.reload();
    }
  }

  onMount(() => {
    // Read initial state from registry
    state = game.registry.get('demoGameState') as DemoGameState ?? null;
    // Subscribe to live updates
    game.events.on(GameEvents.STATE_CHANGED, onStateChanged);
  });

  onDestroy(() => {
    game.events.off(GameEvents.STATE_CHANGED, onStateChanged);
  });
</script>

<div class="fixed top-4 left-4 z-50 w-80 max-h-[80vh] overflow-y-auto
            bg-gray-950/85 border border-gray-700 rounded-lg p-3
            font-mono text-xs text-gray-300 pointer-events-auto select-text">
  <div class="flex items-center justify-between mb-2">
    <span class="text-amber-400 font-bold text-sm">QA MODE</span>
    <span class="text-gray-500 text-[10px]">?qa</span>
  </div>

  {#if state}
    <div class="space-y-1.5">
      <!-- Map & Position -->
      <div>
        <span class="text-gray-500">Map:</span>
        <span class="text-teal-400">{state.currentMap}</span>
      </div>
      <div>
        <span class="text-gray-500">Pos:</span>
        <span>({Math.round(state.position.x)}, {Math.round(state.position.y)})</span>
        <span class="text-gray-500 ml-1">Face:</span>
        <span>{state.facing}</span>
      </div>

      <!-- XP -->
      <div>
        <span class="text-gray-500">XP:</span>
        <span class="text-amber-300">{state.xp}</span>
      </div>

      <!-- Booleans -->
      <div class="flex flex-wrap gap-1">
        <span class={state.introSeen ? 'text-green-400' : 'text-gray-600'}>intro</span>
        <span class={state.terminalUnlocked ? 'text-green-400' : 'text-gray-600'}>terminal</span>
        <span class={state.mapUnlocked ? 'text-green-400' : 'text-gray-600'}>map</span>
      </div>

      <!-- Quests -->
      <div>
        <span class="text-gray-500">Quest:</span>
        <span class={state.quests.active ? 'text-teal-400' : 'text-gray-600'}>
          {state.quests.active ?? 'none'}
        </span>
      </div>
      {#if state.quests.completed.length > 0}
        <div>
          <span class="text-gray-500">Done:</span>
          <span class="text-green-400">{state.quests.completed.join(', ')}</span>
        </div>
      {/if}

      <!-- Flags -->
      <div>
        <div class="text-gray-500 mb-0.5">Flags ({state.flags.length}):</div>
        {#if state.flags.length > 0}
          <div class="flex flex-wrap gap-1">
            {#each state.flags as flag}
              <span class="bg-gray-800 px-1 rounded text-[10px] text-gray-400">{flag}</span>
            {/each}
          </div>
        {:else}
          <span class="text-gray-600">none</span>
        {/if}
      </div>
    </div>

    <!-- Actions -->
    <div class="mt-3 pt-2 border-t border-gray-700">
      <button
        on:click={resetState}
        class="w-full px-2 py-1.5 bg-red-900/60 hover:bg-red-800/80 border border-red-700/50
               rounded text-red-300 text-xs font-bold cursor-pointer transition-colors"
      >
        Reset State
      </button>
    </div>
  {:else}
    <div class="text-gray-600 italic">Waiting for game state...</div>
  {/if}
</div>
```

#### 2. Wire QA Overlay in PhaserGame.svelte

**File**: `src/explorers/PhaserGame.svelte`

Add URL parameter detection and conditionally render the QA overlay:

```svelte
// In <script> section, add:
import QaOverlay from './QaOverlay.svelte';

let qaMode = false;

// In onMount, before game creation:
qaMode = new URLSearchParams(window.location.search).has('qa');

// In template, add after SmartTerminal overlay:
{#if qaMode && game}
  <QaOverlay {game} />
{/if}
```

## Performance Considerations

- The QA overlay subscribes to a single event (`STATE_CHANGED`) which already fires on every state mutation — no additional overhead.
- The panel is a lightweight DOM element (~20 elements) outside the Phaser canvas — negligible rendering cost.
- No polling, no timers, no additional localStorage reads.
