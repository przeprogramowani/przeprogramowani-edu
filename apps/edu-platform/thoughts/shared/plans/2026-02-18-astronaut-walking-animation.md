# Astronaut Walking Animation Implementation Plan

## Overview

Extend the astronaut sprite from a 4-frame static strip (one frame per direction) to a 16-frame animated spritesheet (4 walk-cycle frames × 4 directions), with Phaser walk/idle animations triggered by the existing FSM states.

## Current State Analysis

- **Sprite**: Single row, 4 frames (128×48px). Each frame is a colored rectangle with a directional arrow. Generated by `scripts/generate-placeholder-assets.mjs:306-357`.
- **Loading**: `PreloaderScene.ts:50-53` loads as spritesheet with `frameWidth: 32, frameHeight: 48`.
- **Entity**: `Astronaut.ts` uses `setFrame(FACING_FRAMES[dir])` for static frame selection. The `FACING_FRAMES` map is `{ down: 0, left: 1, right: 2, up: 3 }`.
- **FSM**: Already tracks `'walking'` and `'idle'` states but they don't trigger any animation playback.
- **No Phaser animations exist**: Zero `anims.create()` calls in the codebase.

### Key Discoveries:

- `Astronaut.ts:91-104` — FSM transitions between `walking` and `idle` are already correctly triggered based on velocity, but only static frames are set
- `PreloaderScene.ts:62-76` — `create()` method exists and currently just starts GameScene; this is where we'll register animations
- `constants.ts:3-4` — `PLAYER_FRAME_WIDTH = 32`, `PLAYER_FRAME_HEIGHT = 48` already defined
- The `sprites-theme.md` spec defines a 4-col × 4-row walk grid (matching our target) plus a 5th emote row (out of scope for now)

## Desired End State

The astronaut visually animates when walking: legs alternate left/right as the player moves in any of the 4 directions. When the player stops, the astronaut displays a static idle frame facing the last direction of movement.

### Verification:
- Moving in any direction shows a looping 4-frame walk cycle
- Stopping movement shows a static idle frame matching the facing direction
- Diagonal movement plays the animation for the dominant axis direction
- State transitions (entering dialogue, cutscene) stop the animation and show idle frame
- The placeholder sprite clearly shows leg movement across frames

## What We're NOT Doing

- Row 5 emote/interaction frames (waving, examining, breathing, terminal)
- Replacing placeholder art with real pixel art
- Adding animation speed modifiers (e.g., running)
- Per-pixel collision changes
- AnimationRegistry abstraction (keeping it simple in PreloaderScene)

## Implementation Approach

Three-phase approach: generate new sprite → register animations → wire to entity logic. Each phase is independently testable.

## Critical Implementation Details

### Sprite Sheet Layout

New layout: **4 columns × 4 rows** = 16 frames, sheet size **128 × 192 px**.

```
Row 0 (down):   [idle(0)]  [step-L(1)]  [idle(2)]  [step-R(3)]
Row 1 (left):   [idle(4)]  [step-1(5)]  [idle(6)]  [step-2(7)]
Row 2 (right):  [idle(8)]  [step-1(9)]  [idle(10)] [step-2(11)]
Row 3 (up):     [idle(12)] [step-L(13)] [idle(14)]  [step-R(15)]
```

Phaser frame indices (left-to-right, top-to-bottom): 0-3 = down, 4-7 = left, 8-11 = right, 12-15 = up.

### Animation Definitions

```typescript
// Walk animations — 4 frames per direction, looping
{ key: 'walk-down',  frames: [0, 1, 2, 3],   frameRate: 8, repeat: -1 }
{ key: 'walk-left',  frames: [4, 5, 6, 7],   frameRate: 8, repeat: -1 }
{ key: 'walk-right', frames: [8, 9, 10, 11], frameRate: 8, repeat: -1 }
{ key: 'walk-up',    frames: [12, 13, 14, 15], frameRate: 8, repeat: -1 }
```

Frame rate of 8 fps at `PLAYER_SPEED = 120` gives a natural-feeling walk (~15px per frame step).

### Idle Frame Mapping

```typescript
const IDLE_FRAMES: Record<FacingDirection, number> = {
  down: 0,
  left: 4,
  right: 8,
  up: 12,
};
```

### Placeholder Leg Markers

Each 32×48 frame keeps the existing teal body + white arrow. Leg markers are drawn at the bottom of the sprite (y=38..44 area):

- **Idle frames (col 0, 2)**: Two small rectangles centered — neutral stance
- **Step-left frames (col 1)**: Left foot rectangle shifted forward (lower), right foot back
- **Step-right frames (col 3)**: Right foot rectangle shifted forward, left foot back

For left/right-facing rows, "forward" means along the horizontal axis instead of vertical.

### Timing & Lifecycle Considerations

**N/A for complex timing**: Phaser's `anims.play()` with `repeat: -1` handles the walk loop. `stop()` + `setFrame()` handles idle. No custom timers or lifecycle hooks needed.

### Performance & Optimization Strategy

**N/A**: The animation system is built into Phaser's sprite class. No additional memoization or optimization needed for a single animated sprite.

- Verify by moving the astronaut in all 4 directions and observing frame changes
- Check Phaser's animation manager: `this.anims.get('walk-down')` should exist after create()

## Phase 1: Update Placeholder Sprite Generator

### Overview

### Changes Required:

#### 1. `scripts/generate-placeholder-assets.mjs`

**File**: `scripts/generate-placeholder-assets.mjs`
**Changes**: Rewrite `generateAstronaut()` to produce a 4-column × 4-row grid with leg markers.

```javascript
function generateAstronaut() {
  const W = 32;
  const H = 48;
  const COLS = 4;  // frames per direction
  const ROWS = 4;  // directions: down, left, right, up
  const canvas = createCanvas(W * COLS, H * ROWS);
  const ctx = canvas.getContext('2d');

  // Direction arrows (same as before, one per row)
  const arrows = [
    // down ▼
    (cx, cy) => { ctx.moveTo(cx - 4, cy - 3); ctx.lineTo(cx + 4, cy - 3); ctx.lineTo(cx, cy + 4); },
    // left ◀
    (cx, cy) => { ctx.moveTo(cx + 3, cy - 4); ctx.lineTo(cx + 3, cy + 4); ctx.lineTo(cx - 4, cy); },
    // right ▶
    (cx, cy) => { ctx.moveTo(cx - 3, cy - 4); ctx.lineTo(cx - 3, cy + 4); ctx.lineTo(cx + 4, cy); },
    // up ▲
    (cx, cy) => { ctx.moveTo(cx - 4, cy + 3); ctx.lineTo(cx + 4, cy + 3); ctx.lineTo(cx, cy - 4); },
  ];

  // Leg patterns per column: [leftFootOffsetY, rightFootOffsetY]
  // 0 = neutral, negative = foot forward (down), positive = foot back (up)
  const legPatterns = [
    [0, 0],    // col 0: idle — both feet neutral
    [-3, 3],   // col 1: step-left — left foot forward, right back
    [0, 0],    // col 2: idle — both feet neutral
    [3, -3],   // col 3: step-right — right foot forward, left back
  ];

  for (let row = 0; row < ROWS; row++) {
    for (let col = 0; col < COLS; col++) {
      const x = col * W;
      const y = row * H;

      // Body — teal rectangle
      ctx.fillStyle = '#00d4aa';
      ctx.fillRect(x + 4, y + 4, 24, 34);

      // Arrow — white directional indicator
      ctx.fillStyle = '#ffffff';
      ctx.beginPath();
      arrows[row](x + W / 2, y + 14);
      ctx.closePath();
      ctx.fill();

      // Legs — small rectangles at bottom
      const [leftOff, rightOff] = legPatterns[col];
      const legY = y + 38;
      const legW = 6;
      const legH = 8;

      // Left leg
      ctx.fillStyle = '#009977';
      ctx.fillRect(x + 7, legY + leftOff, legW, legH);

      // Right leg
      ctx.fillStyle = '#009977';
      ctx.fillRect(x + 19, legY + rightOff, legW, legH);
    }
  }

  const outPath = join(PUBLIC, 'sprites', 'placeholder-astronaut.png');
  ensureDir(outPath);
  writeFileSync(outPath, canvas.toBuffer('image/png'));
  console.log(`✓ ${outPath} (${COLS}x${ROWS} = ${COLS * ROWS} frames)`);
}
```

## Phase 3: Wire Animations to Astronaut Entity

### Overview

Replace static `setFrame()` calls with Phaser animation `play()`/`stop()` based on the FSM walking/idle states.

### Changes Required:

#### 1. Astronaut Entity Animation Logic

**File**: `src/explorers/entities/Astronaut.ts`
**Changes**: Modify `setFacing()` and the update loop to play/stop animations.

The key change: when the astronaut is walking, play the direction-specific walk animation. When idle (or in dialogue/cutscene/interacting), stop the animation and show the static idle frame.

```typescript
// Add animation key helper
private getWalkAnimKey(dir: FacingDirection): string {
  return `walk-${dir}`;
}

setFacing(dir: FacingDirection): void {
  this._facing = dir;
  // Only set static frame if NOT walking (animation handles frames during walk)
  if (!this.fsm.isState('walking')) {
    this.setFrame(FACING_FRAMES[dir]);
  }
}

update(): void {
  if (this.fsm.isState('dialogue') || this.fsm.isState('cutscene') || this.fsm.isState('interacting')) {
    (this.body as Phaser.Physics.Arcade.Body).setVelocity(0, 0);
    this.stop();  // Stop any running animation
    return;
  }

  let vx = 0;
  let vy = 0;

  if (this.inputCtrl.isLeft()) vx -= 1;
  if (this.inputCtrl.isRight()) vx += 1;
  if (this.inputCtrl.isUp()) vy -= 1;
  if (this.inputCtrl.isDown()) vy += 1;

  // Normalize diagonal movement
  if (vx !== 0 && vy !== 0) {
    const norm = Math.SQRT1_2;
    vx *= norm;
    vy *= norm;
  }

  const body = this.body as Phaser.Physics.Arcade.Body;
  body.setVelocity(vx * PLAYER_SPEED, vy * PLAYER_SPEED);

  // Update facing direction and animation
  if (vx !== 0 || vy !== 0) {
    if (!this.fsm.isState('walking')) {
      this.fsm.transition('walking');
    }

    // Determine facing
    let newFacing: FacingDirection;
    if (Math.abs(vx) >= Math.abs(vy)) {
      newFacing = vx < 0 ? 'left' : 'right';
    } else {
      newFacing = vy < 0 ? 'up' : 'down';
    }

    // Play walk animation (only restart if direction changed)
    const animKey = this.getWalkAnimKey(newFacing);
    if (this._facing !== newFacing || !this.anims.isPlaying) {
      this._facing = newFacing;
      this.play(animKey, true);
    }
  } else {
    if (this.fsm.isState('walking')) {
      this.fsm.transition('idle');
    }
    // Stop animation, show idle frame
    this.stop();
    this.setFrame(FACING_FRAMES[this._facing]);
  }
}
```

Key behavioral details:
- `this.play(animKey, true)` — the `true` parameter means "ignore if this animation is already playing", preventing restarts on every frame
- When direction changes mid-walk, the animation restarts for the new direction
- `this.stop()` halts the current animation and `setFrame()` shows the idle pose
- The `setFacing()` method now only sets the static frame when NOT walking (during walking, the animation handles frame selection)

#### 2. Remove `getWalkAnimKey` if preferred inline

The helper is tiny — could be inlined as `` `walk-${dir}` `` directly. Either approach works.
