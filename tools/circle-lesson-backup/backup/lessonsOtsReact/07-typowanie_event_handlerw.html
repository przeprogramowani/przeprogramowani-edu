Fundamentem nowoczesnych aplikacji webowych jest obsługa interakcji użytkownika z interfejsem.

React z TypeScript oferuje wiele możliwości zarządzania tymi interakcjami w bezpieczny sposób. Poprawne typowanie event handlerów nie tylko zwiększa bezpieczeństwo kodu, ale również ułatwia jego utrzymanie i rozwój.

W tej lekcji omówimy różne rodzaje zdarzeń w React, pokażemy, jak efektywnie typować event handlerów przy użyciu TypeScript oraz przedstawimy najlepsze praktyki, które pomogą unikać typowych błędów.

Rodzaje eventów

React obsługuje szeroki zakres zdarzeń, które odpowiadają różnym interakcjom użytkownika z aplikacją. Poniżej przedstawiamy listę najważniejszych typów eventów dostępnych w React 18+:

  • AnimationEvent – związany z animacjami CSS, pozwala reagować na różne etapy animacji.

  • ChangeEvent – występuje przy zmianie wartości elementów <input>, <select> oraz <textarea>, co jest kluczowe przy obsłudze formularzy.

  • ClipboardEvent – obejmuje zdarzenia związane ze schowkiem, takie jak kopiowanie, wklejanie czy wycinanie tekstu.

  • CompositionEvent – zdarzenia, które występują, gdy użytkownik wprowadza tekst w sposób niebezpośredni, na przykład pisząc po japońsku na polskiej klawiaturze za pomocą Input Method Editor (IME).

  • DragEvent – obsługuje interakcje typu "drag and drop" wykonywane myszką.

  • FocusEvent – wywoływany, gdy element zyskuje lub traci fokus, co jest istotne dla dostępności aplikacji.

  • FormEvent – pojawia się za każdym razem, gdy formularz lub jego element uzyskuje lub traci fokus, wartość elementu zostaje zmieniona lub formularz zostaje przesłany.

  • InvalidEvent – uruchamiany, gdy kontrolka formularza nie przechodzi walidacji, na przykład gdy użytkownik wprowadzi liczbę przekraczającą dozwolony zakres w polu <input type="number" max="10">.

  • KeyboardEvent – odpowiada interakcjom z klawiaturą, gdzie każdy event odpowiada naciskaniu pojedynczego klawisza.

  • MouseEvent – dotyczy interakcji użytkownika z myszką.

  • PointerEvent – obejmuje interakcje z dowolnym urządzeniem wskazującym, takim jak myszka, rysik czy ekran dotykowy. Jest rekomendowanym rozwiązaniem zamiast MouseEvent ze względu na większą uniwersalność.

  • TouchEvent – zdarzenia wynikające z interakcji użytkownika z urządzeniem dotykowym, rozszerzające UIEvent.

  • TransitionEvent – związany z przejściami CSS, umożliwia reagowanie na ich zakończenie.

  • UIEvent – bazowy typ dla wielu innych eventów, takich jak MouseEvent, TouchEvent czy PointerEvent.

  • WheelEvent – obsługuje scrollowanie za pomocą kółka myszy lub podobnych urządzeń wejściowych, odpowiada za event typu wheel, a nie scroll.

  • SyntheticEvent – podstawowy typ dla wszystkich powyższych eventów w React, zapewniający spójne zachowanie niezależnie od przeglądarki.

Jak typować w praktyce?

W codziennej pracy często napotkasz sytuacje, w których TypeScript nie poradzi sobie z automatycznym określeniem typu parametru eventu. Na przykład:

const handleClick = (e) => {}; // Parameter 'e' implicitly has an 'any' type.

<button onClick={handleClick}>Click me</button>;

W powyższym przykładzie TypeScript zgłasza błąd, ponieważ nie jest jasne, jaki typ powinien mieć parametr e w funkcji handleClick. Problem ten pojawia się również w przypadku innych funkcji obsługi zdarzeń, takich jak onSubmit czy onMouseOver, które otrzymują różne elementy DOM jako argumenty.

Aby poprawnie typować zdarzenia, warto zastosować pewne techniki ułatwiające określenie właściwego typu. Jednym z podejść jest stworzenie funkcji inline wewnątrz obsługi zdarzenia, co pozwala na automatyczne rozpoznanie typu przez TypeScript:

<button onClick={(e) => {}} /> // (parametr) e: React.MouseEvent<HTMLButtonElement>

Po najechaniu kursorem na e, TypeScript wskazuje, że ma on typ React.MouseEvent<HTMLButtonElement>. Możemy teraz skopiować ten typ i użyć go do zdefiniowania naszej funkcji handleClick:

const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
  console.log(e);
};

const ExampleComponent = () => (
  <button onClick={handleClick}>
    Click me
  </button>
);

Chociaż ten sposób działa, może wydawać się nieco powolny, zwłaszcza gdy mamy do czynienia z wieloma eventami w aplikacji. Istnieje jednak bardziej efektywna metoda, która pozwala na automatyczne wywnioskowanie typu bez konieczności ręcznego kopiowania go.

#React.ComponentProps

Sposobem na przyspieszenie tego byłoby usunięcie kroku, w którym sprawdzamy typ obsługi. Byłoby wspaniale powiedzieć „Chcę tego typu obsługi” i pozwolić TypeScriptowi dowiedzieć się reszty.

W tym celu możemy użyć ComponentProps, który omówiliśmy szczegółowo w lekcji #Ekstrakcja propsów. Możemy go wykorzystać również w tym kontekście, zawężając działanie do typu event handlera:

const handleClick: React.ComponentProps<"button">["onClick"] = (e) => {
  console.log(e);
};

const ExampleButton = () => (
  <button onClick={handleClick}>
    Click me
  </button>
);

W powyższym przykładzie, ComponentProps<"button"> zwraca typ propsów dla elementu <button>, a ["onClick"] wyciąga typ właściwości onClick. Dzięki temu TypeScript automatycznie określa, że e ma typ React.MouseEvent<HTMLButtonElement>, co eliminuje potrzebę ręcznego definiowania typu.

Eventy natywne vs eventy Reacta

Podczas pracy z Reactem, warto zrozumieć różnicę między natywnymi eventami JavaScript a eventami dostarczanymi przez Reacta. Na przykład, istnieje zarówno globalny MouseEvent, jak i MouseEvent eksportowany przez Reacta. Oba typy są używane do nasłuchiwania zdarzeń, jednak mają swoje specyficzne cechy.

Natywny MouseEvent odnosi się do standardowych zdarzeń JavaScript i jest częścią globalnego API przeglądarki. Z kolei React.MouseEvent jest dostosowaną wersją tego eventu, zoptymalizowaną pod kątem systemu eventów syntetycznych Reacta. Eventy Reacta uwzględniają kompatybilność między różnymi przeglądarkami i zawierają dodatkowe właściwości specyficzne dla Reacta, takie jak persist.

Choć oba typy eventów mają wiele wspólnych cech i mogą być używane zamiennie w wielu przypadkach, zaleca się korzystanie z eventów Reacta, aby w pełni wykorzystać optymalizacje i dodatkowe funkcjonalności oferowane przez bibliotekę.

Dołączanie natywnych event listenerów

Czasami w aplikacji React potrzebujemy dodać natywne event listenery, które nie są bezpośrednio związane z konkretnymi komponentami. Przykładem może być globalny listener na zdarzenie scroll, który reaguje na przewijanie całej strony.

W takich przypadkach zazwyczaj używamy hooka useEffect do rejestracji i usunięcia event listenera:

function Navigation() {
  useEffect(() => {
    const onScroll = (e: Event) => {
      // Obsługa zdarzenia przewijania
      console.log('Scrolling', e);
    };

    document.addEventListener('scroll', onScroll);
    return () => {
      document.removeEventListener('scroll', onScroll);
    };
  }, []);

  return (
    // Logika komponentu
    <nav>
      {/* Elementy nawigacyjne */}
    </nav>
  );
}

W powyższym przykładzie, onScroll jest typowany jako Event, ponieważ jest to natywny event przeglądarki. Ponieważ ten listener jest niezależny od systemu eventów Reacta, używamy natywnych typów, aby zapewnić poprawność typowania.

Używaj event.currentTarget zamiast event.target

Podczas obsługi zdarzeń w React warto korzystać z event.currentTarget zamiast event.target, aby uniknąć potencjalnych błędów wynikających z delegacji zdarzeń:

const UnsafeExample = () => {
  const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
    // ❌ Niebezpieczne - może rzucić błędem w runtime dla img oraz span
    const value = e.target.value; 
  };

  return (
    <button onClick={handleClick}>
      <span>Click me</span>
      <img src="icon.png" alt="" />
    </button>
  );
};

event.currentTarget odnosi się do elementu, do którego przypisano listener, podczas gdy event.target może odnosić się do dowolnego elementu w drzewie DOM, na którym faktycznie doszło do zdarzenia.

const SafeExample = () => {
  const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
    // Bezpieczne - currentTarget jest zawsze HTMLButtonElement
    const button = e.currentTarget;
    console.log(button.value);
  };

  return (
    <button onClick={handleClick}>
      <span>Click me</span>
      <img src="icon.png" alt="" />
    </button>
  );
};

Unikaj korzystania z event.target bez odpowiednich zabezpieczeń

Korzystanie z event.target może prowadzić do niebezpiecznych sytuacji, gdy użytkownik kliknie w element wewnątrz komponentu, który nie ma oczekiwanych właściwości. Aby bezpiecznie korzystać z event.target, należy stosować type guardy, które sprawdzają typ elementu przed jego użyciem:

const SafeClickExample = () => {
  const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
    if (e.target instanceof HTMLSpanElement) {
      // Obsługa kliknięcia w <span>
      console.log('Clicked on span:', e.target.textContent);
    } else if (e.target instanceof HTMLImageElement) {
      // Obsługa kliknięcia w <img>
      console.log('Clicked on image:', e.target.src);
    } else {
      // Obsługa kliknięcia w <button>
      console.log('Clicked on button');
    }
  };

  return (
    <button onClick={handleClick}>
      <span>Click</span>
      <img src="icon.png" alt="icon" />
    </button>
  );
};

W tym przykładzie, używając instanceof, upewniamy się, że e.target jest odpowiednim typem przed dostępem do jego właściwości, co zapobiega błędom w czasie wykonywania.

Podsumowanie

Typowanie event handlerów w React z wykorzystaniem TypeScript jest kluczowe dla zapewnienia bezpieczeństwa typów i uniknięcia błędów w aplikacji.

Korzystając z takich narzędzi jak ComponentProps, możemy uprościć proces typowania i uczynić nasz kod bardziej czytelnym i mniej podatnym na błędy.

Ważne jest również zrozumienie różnicy między natywnymi eventami a eventami Reacta oraz stosowanie dobrych praktyk, takich jak preferowanie event.currentTarget nad event.target.
