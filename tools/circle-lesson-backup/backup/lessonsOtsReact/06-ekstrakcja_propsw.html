Jednym z najużyteczniejszych helperów typu eksportowanych przez bibliotekę React jest ComponentProps. Ten generyk umożliwia ekstrakcję typu propsów z istniejącego komponentu lub elementu HTML.

Dzięki temu można łatwo ponownie wykorzystać typy oraz zapewnić, że są one aktualizowane dynamicznie, eliminując potrzebę manualnego przypisywania i utrzymywania zduplikowanych typów.

Ekstrakcja propsów z ComponentProps

Jednym z najużyteczniejszych helperów typu eksportowanych przez bibliotekę React jest ComponentProps.

React.ComponentProps to typ generyczny, które przyjmuje komponent lub nazwę element HTML (jako string) i zwraca typ jego propsów.

Dzięki temu można łatwo ponownie wykorzystać typy oraz zapewnić, że są one aktualizowane dynamicznie, eliminując potrzebę manualnego przypisywania i utrzymywania zduplikowanych typów.

import { ComponentProps } from 'react';

type ButtonProps = ComponentProps<'button'>;

const Button = (props: ButtonProps) => (
  <button {...props}>Click me</button>
);

W powyższym przykładzie ButtonProps ekstraktuje typy propsów z natywnego elementu HTML <button>. Dzięki temu komponent Button dziedziczy wszystkie standardowe właściwości, takie jak disabled czy onClick, co zwiększa jego elastyczność i odporność na ewentualne zmiany/nowości.

Duplikowanie propsów innych komponentów

React.ComponentProps umożliwia tworzenie komponentów, które automatycznie dziedziczą typy z innych komponentów lub elementów, co minimalizuje potrzebę ręcznego zarządzania typami.

import { ComponentProps } from 'react';

const CustomInput = ({ value }: { value: string }) => <input value={value} />;

type CustomInputProps = ComponentProps<typeof CustomInput>;

const FormField = (props: CustomInputProps) => <CustomInput {...props} />;

W tym przypadku FormField przyjmuje te same propsy co CustomInput, co zapewnia spójność typów bez konieczności ich ręcznego przepisywania.

ComponentPropsWithRef - Typowanie z obsługą Ref

ComponentPropsWithRef to rozwinięcie ComponentProps, które pozwala na dodanie obsługi refów. Jest to kluczowe, gdy potrzebujemy przekazać ref, który będzie używany przez komponent. Dzięki temu ref jest traktowany jako integralna część komponentu.

import { ComponentPropsWithRef } from 'react';

type InputWithRefProps = ComponentPropsWithRef<'input'>; //

const InputWithRef = React.forwardRef<HTMLInputElement, InputWithRefProps>((props, ref) => (
  <input ref={ref} {...props} />
));

W powyższym przykładzie komponent InputWithRef korzysta z ComponentPropsWithRef<'input'>, co oznacza, że oprócz wszystkich typowych propsów <input>, obsługuje również ref do tego elementu. Dzięki temu można łatwo zarządzać focusem i innymi właściwościami elementu poprzez ref.

React.JSX.IntrinsicElements – Alternatywa dla HTML Elementów

React.JSX.IntrinsicElements to typ, który podobnie jak ComponentProps, pozwala na odziedziczenie typów natywnych elementów HTML. Różnica polega na tym, że React.JSX.IntrinsicElements jest używany bezpośrednio dla HTML elementów, zapewniając pełne wsparcie dla natywnych atrybutów, ale bez dodatkowych właściwości Reacta (onClick, ref, key itd.). Z tego względu ComponentProps jest preferowany do komponentów React, ale IntrinsicElements zapewnia większą precyzję dla typowego HTML.

type ButtonIntrinsicProps = React.JSX.IntrinsicElements[`button`];

const CustomButton = (props: ButtonIntrinsicProps) => {
  return <button {...props}>Click me</button>
};

W tym przykładzie CustomButton wykorzystuje wszystkie atrybuty natywne button bez dodatkowych atrybutów Reacta. Jest to przydatne w projektach, gdzie typy Reactowe nie są wymagane, a ważne jest trzymanie się ścisłych typów HTML. W praktyce taki wymóg jest rzadki, stąd rekomendujemy stosowanie ComponentProps dla zachowania spójności w projekcie.

interface extends zamiast operatora & dla złożonych struktur typów

Stosowanie operatora przecięcia (&) w aliasach typów może prowadzić do spowolnienia wydajności w dużych projektach - szczególnie gdy używane są dla złożonych typów, które powstają z ComponentProps (więcej szczegółów w TypeScript Perfomance Wiki)

Dlatego zalecamy używanie interface extends, które jest bardziej wydajnym podejściem:

// Mniej wydajne - złożone przecięcie
type InputProps = ComponentProps<'input'> & {
  label: string;
};

// Bardziej wydajne podejście z `interface extends`
interface InputProps extends ComponentProps<'input'> {
  label: string;
}

const LabeledInput = ({ label, ...props }: InputProps) => (
  <label>
    {label}
    <input {...props} />
  </label>
);

W powyższym przykładzie interface extends umożliwia typowi InputProps rozszerzenie typów propsów elementu input bez negatywnego wpływu na wydajność, co jest bardziej efektywne niż użycie operatora &.

Pick i selektywne wybieranie propsów z ComponentProps

W sytuacjach, gdy nie potrzebujesz wszystkich propsów komponentu, Pick może pomóc ograniczyć odziedziczone propsy tylko do tych, które są wymagane. Dzięki Pick możemy wybrać tylko konkretne właściwości i uniknąć nadmiarowych typów, co poprawia czytelność i kontrolę.

import { ComponentProps } from 'react';

type ProductTileProps = {
  title: string;
  description?: string;
  price: number;
  imageSrc: string;
  onAddToCart: () => void;
};

const ProductTile = ({ title, description, price, imageSrc, onAddToCart }: ProductTileProps) => (
  <div>
    <img src={imageSrc} alt={title} />
    <h2>{title}</h2>
    {description && <p>{description}</p>}
    <p>Price: ${price}</p>
    <button onClick={onAddToCart}>Add to Cart</button>
  </div>
);

// Zdefiniujemy jednak nowy komponent HighlightedProductTile, który potrzebuje tylko `title` i `description`
// Za pomocą `Pick` wybieramy tylko te propsy, które są nam potrzebne
interface HighlightedTileProps extends Pick<ComponentProps<typeof ProductTile>, "title" | "description"> {
  highlightColor: string;
}

const HighlightedProductTile = ({ highlightColor, title, description }: HighlightedTileProps) => (
  <div style={{ border: `2px solid ${highlightColor}`, padding: '1rem' }}>
    <h2>{title}</h2>
    {description && <p>{description}</p>}
  </div>
);

// Użycie komponentu HighlightedProductTile
const App = () => (
  <div>
    <HighlightedProductTile title="Stylish Shoes" description="Comfortable and stylish" highlightColor="blue" />
    {/* `price`, `imageSrc`, i `onAddToCart` są pominięte i nie są wymagane */}
  </div>
);

W tym przykładzie HighlightedProductTile dziedziczy tylko title i description z ProductTile, dzięki czemu komponent jest bardziej czytelny i lepiej kontrolowany.

Podsumowanie lekcji

W tej lekcji omówiliśmy różne techniki ekstrakcji propsów:

  • ComponentProps jest przydatny do ekstrakcji typów propsów zarówno z elementów HTML, jak i komponentów React, co pozwala na elastyczne i dynamiczne zarządzanie typami.

  • ComponentPropsWithRef rozszerza te możliwości o obsługę refów, co jest kluczowe dla komponentów wymagających zarządzania fokusem.

  • React.JSX.IntrinsicElements oferuje alternatywę dla ekstrakcji typów propsów natywnych elementów HTML, choć w praktyce częściej zaleca się stosowanie ComponentProps dla zachowania spójności w projekcie.

  • Dodatkowo, stosowanie interface extends zamiast operatora przecięcia (&) pozwala na bardziej wydajne typowanie w złożonych strukturach, a Pick umożliwia selektywne wybieranie tylko potrzebnych propsów, co poprawia czytelność i kontrolę nad komponentami.

Opanowanie tych technik znacząco zwiększa elastyczność i utrzymywalność kodu w aplikacjach Reactowych, szczególnie w dużych projektach lub przy pracy z bibliotekami komponentów.
