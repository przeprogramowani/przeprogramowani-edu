Higher Order Components (HOC) to zaawansowany wzorzec projektowy w React, który umożliwia ponowne wykorzystanie logiki komponentów poprzez dekorację. HOC to funkcje, które przyjmują komponent jako argument i zwracają nowy komponent z dodatkowymi funkcjonalnościami.

Chociaż hooki przejęły wiele dawnych przypadków użycia Higher Order Components, zwłaszcza tych związanych z zarządzania stanem i obsługą efektów ubocznych, ten wzorzec nadal jest wykorzystywany w wielu bibliotekach i dobrze sprawdza się w aplikacjach w następujących sytuacjach:

  • HOC może weryfikować czy użytkownik jest uwierzytelniony przed renderowaniem lub w celu ograniczenia dostępu na podstawie ról użytkownika.

  • HOC może warunkowo renderować komponenty w oparciu o flagi funkcji lub testy A/B.

  • HOC może zapewnić funkcjonalność tłumaczenia komponentów.

  • HOC może dodawać logowanie lub analitykę do komponentów w transparentny sposób.

Spotkasz się z dwoma rodzajami Higher Order Components:

  • Enhancers, które rozszerzają komponent o nową funkcjonalność

  • Injectors, które wstrzykują wymagane propsy do komponentu

Poniżej omówimy przykłady tych dwóch kategorii wraz z opisem ich typowania.

Przykład enhancera: withAuth

HOC mogą dekorować komponenty w celu sprawdzenia, czy użytkownik jest uwierzytelniony przed renderowaniem, lub ograniczyć dostęp na podstawie ról użytkownika. Dzięki temu można łatwo zarządzać dostępem do różnych części aplikacji bez konieczności modyfikowania samego komponentu:

// src/hoc/with-auth.tsx
import React from "react";
import {Redirect} from "react-router-dom";
import {useAuth} from "../hooks/use-auth";

export function withAuth<P extends object>(
  WrappedComponent: React.ComponentType<P>
) {
  const ComponentWithAuth = (props: P) => {
    const isAuthenticated = useAuth();

    if (!isAuthenticated) {
      return <Redirect to='/login' />;
    }

    return <WrappedComponent {...props} />;
  };

  ComponentWithAuth.displayName = `withAuth(${
    WrappedComponent.displayName || WrappedComponent.name || "Component"
  })`; // Ustawianie czytelnego displayName dla React DevTools

  return ComponentWithAuth;
}

// src/components/UserProfile.tsx
function UserProfile {
 // zawartość komponentu
}

const UserProfileWithAuth = withAuth(UserProfile)

export default UserProfileWithAuth;

Typowanie enhancera withAuth wygląda następująco:

  1. Parametr generyczny typu P:

    1. P extends object oznacza, że P jest typem obiektowym, co pozwala na przekazanie dowolnych propsów.

    2. Dzięki temu withAuth może być używany z dowolnym komponentem.

  2. Typ React.ComponentType<P>:

    1. WrappedComponent: React.ComponentType<P> oznacza, że WrappedComponent jest komponentem React, który przyjmuje propsy typu P.

    2. React.ComponentType<P> to unia typów, która obejmuje zarówno komponenty funkcyjne, jak i klasowe.

  3. Zwracany typ ComponentWithAuth:

    1. ComponentWithAuth jest komponentem funkcyjnym, który również przyjmuje propsy typu P.

    2. ComponentWithAuth opakowuje Redirect do strony logowania, jeśli użytkownik nie jest uwierzytelniony, lub WrappedComponent z przekazanymi propsami, jeśli użytkownik jest uwierzytelniony.

  4. displayName - ComponentWithAuth.displayName jest ustawiany dla lepszej czytelności w React DevTools, co ułatwia debugowanie.

Przykład injectora: withTranslation

HOC mogą zapewniać funkcje tłumaczenia i internacjonalizacji komponentom poprzez wstrzykiwanie funkcji tłumaczenia i zarządzanie ustawieniami regionalnymi. Ten wzorzec pozwala komponentom pozostać niezależnymi od języków, podczas gdy logika tłumaczenia jest scentralizowana i wielokrotnego użytku:

export interface TranslationProps {
  t: (key: string, params?: Record<string, string | number>) => string;
  currentLocale: string;
}

type Subtract<T, K> = Omit<T, keyof K>;

// Translation HOC
export const withTranslation = <P extends TranslationProps>(
  WrappedComponent: React.ComponentType<P>
) => {
  return function WithTranslation(props: Subtract<P, TranslationProps>) {
    const [dictionary, setDictionary] = useState<Record<string, string>>({});
    const [locale, setLocale] = useState('en');

    const loadTranslations = async (locale: string) => {
      const response = await fetch(`/api/translations/${locale}`);
      setDictionary(await response.json());
    };

    useEffect(() => {
      loadTranslations(locale);
    }, [locale]);

    const translate = useCallback((key: string, params?: Record<string, string | number>): string => {
      const template = dictionary[key] || key;
      if (!params) return template;
      
      return Object.entries(params).reduce(
        (text, [param, value]) => text.replace(`{{${param}}}`, String(value)),
        template
      );
    }, [dictionary]);

    const injectedProps = {
      t: translate,
      currentLocale: locale
    } as TranslationProps;

    return <WrappedComponent {...props} {...injectedProps} />;
  };
};

// Example usage showing language-agnostic component
interface GreetingProps extends TranslationProps {
  name: string;
}

const Greeting = ({ name, t, currentLocale }): GreetingProps => (
  <div>
    <h1>{t('greeting', { name })}</h1>
    <p>{t('current_locale')}: {currentLocale}</p>
  </div>
);

// Component wrapped with translation capabilities
const TranslatedComponent = withTranslation(MyComponent);

// Usage without needing to handle translations directly
<TranslatedComponent name="John" />

Typowanie injectora withTranslation wygląda następująco:

  1. Podstawowy interfejs tłumaczenia:

    1. TranslationProps definiuje wstrzykiwany interfejs tłumaczenia

    2. t to funkcja, która przyjmuje klucz w słowniku tłumaczeń i zwraca wartość będącą przypisanym tłumaczeniem

    3. currentLocale: ciąg znaków reprezentujący aktywny język (np. “pl”)

  2. Typy manipulacji propsami:

    1. Subtract<T, K> usuwa propsy tłumaczenia z propsów komponentów.

    2. Subtract wykorzystuje wbudowany utility type Omit

    3. W ten sposób zapobiegamy duplikowaniu deklaracji i przekazywania propsów

  3. Generyk P:

    1. P extends TranslationProps wymusza akceptację propsów translacji przez wrappowany komponent

    2. React.ComponentType<P> zezwala na komponenty klasowe/funkcjonalne

    3. Generyczny typ P przepływa przez całą implementację HOC

    4. Taka konstrukcja utrzymuje informacje o typie dla propsów niezwiązanych z translacją

  1. Zwracany typy funkcji:

    1. Wyklucza propsy związane z tłumaczeniem z zewnętrznego interfejsu komponentu

    2. Wnioskowanie poprawnie wskazuje bazowe propsy komponetów

    3. Wstrzyknięte propsy są dodawane za pomocą asercji as TranslationProps

  2. Typy we wrappowanych komponentach:

    1. Interfejs propsów komponentu rozszerza TranslationProps udostępniany przez HOC

    2. Bazowe propsy komponenty są łączone z propsami tłumaczenia za pomocą przecięcia (&)

    3. Sprawdzanie typu odbywa się zarówno na poziomie HOC, jak i na poziomie użycia

    4. TypeScript wymusza poprawne użycie propsów we wrappowanych komponentach

Podsumowanie

Higher Order Components (HOC) to zaawansowany wzorzec projektowy w React, który pozwala na ponowne wykorzystanie logiki komponentów poprzez ich dekorację, gdzie funkcja przyjmuje komponent jako argument i zwraca nowy komponent z dodatkowymi funkcjonalnościami.

Mimo że hooki przejęły wiele tradycyjnych zastosowań HOC, wzorzec ten nadal jest przydatny w sytuacjach takich jak zarządzanie autoryzacją, testy A/B, tłumaczenia czy dodawanie analityki.

HOC występują w dwóch głównych wariantach: enhancery, które rozszerzają funkcjonalność komponentu, oraz injektory, które wstrzykują do niego dodatkowe propsy.

Dzięki odpowiedniemu typowaniu w TypeScript, HOC zapewniają bezpieczną pod względem typów kompozycję i wielokrotne wykorzystanie kodu, co sprawia, że wciąż są cenną techniką w nowoczesnym rozwoju aplikacji React.
