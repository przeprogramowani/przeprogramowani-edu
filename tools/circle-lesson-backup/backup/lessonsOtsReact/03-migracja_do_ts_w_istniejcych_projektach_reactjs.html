Migracja istniejącego projektu React z JavaScript do TypeScript jest procesem wymagającym systematycznego podejścia i starannego planowania.

W tej lekcji przedstawimy sprawdzone strategie migracji na przykładzie rzeczywistych projektów, w tym studium przypadku firmy Sentry, która skutecznie przemigrowała codebase zawierający ponad 1100 plików.

Skupimy się na konfiguracji TypeScript w projektach React oraz omówimy kluczowe aspekty tego procesu.

Przygotowanie projektu do migracji

Pierwszym krokiem w migracji jest dokładna analiza obecnego stanu aplikacji. Należy przeprowadzić inwentaryzację wszystkich komponentów oraz zależności, co pozwoli zidentyfikować potencjalne wyzwania, takie jak użycie przestarzałych API.

Warto także przeprowadzić audyt zewnętrznych bibliotek, aby sprawdzić, czy są dostępne typy TypeScript dla używanych przez nas paczek.

Następnie należy skonfigurować środowisko pracy, instalując niezbędne pakiety TypeScript oraz typy dla React. Wykonuje się to za pomocą polecenia:

npm install --save-dev typescript @types/react @types/react-dom

Po instalacji TypeScript należy utworzyć podstawowy plik konfiguracyjny tsconfig.json. Przykładowa konfiguracja może wyglądać następująco:

{
  "compilerOptions": {
    "allowJs": true,
    "checkJs": false,
    "jsx": "react-jsx",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "noEmit": true,
    "isolatedModules": true,
    "target": "ES2020"
  },
  "include": ["src/**/*"]
}

Ta konfiguracja pozwala na stopniowe dodawanie TypeScript do istniejącego kodu JavaScript, co jest kluczowe dla płynnej migracji.

Strategie migracji

Istnieją dwie główne strategie migracji: strategia inkrementalna oraz strategia big-bang.

Strategia inkrementalna, którą polecamy dla większości projektów, polega na stopniowym dodawaniu TypeScript do istniejącego kodu. Pozwala to na współistnienie plików .js(x) i .ts(x) w tym samym projekcie, co ułatwia zarządzanie procesem migracji. Priorytetyzacja konwersji powinna opierać się na częstotliwości zmian w komponencie, jego krytyczności biznesowej oraz liczbie zależności.

Z drugiej strony, strategia big-bang polega na jednorazowej konwersji całego kodu. Jest to podejście ryzykowne, ponieważ zwiększa prawdopodobieństwo wystąpienia błędów i przestojów, a także jest trudniejsze do zarządzania w zespole.

Proces migracji krok po kroku

Migracja powinna być przeprowadzana etapami. Pierwszym krokiem jest identyfikacja "leaf components", czyli komponentów bez zależności, które można łatwo skonwertować na TypeScript. Na przykład, prosty komponent Button w JavaScript:

const Button = ({ onClick, children }) => (
  <button onClick={onClick}>{children}</button>
);

Po migracji do TypeScript będzie on wyglądał następująco:

interface ButtonProps {
  onClick: () => void;
  children: React.ReactNode;
}

const Button = ({ onClick, children }: ButtonProps) => (
  <button onClick={onClick}>{children}</button>
);

Kolejnym krokiem jest migracja współdzielonych typów. Przykładowo, w pliku src/@types/common.ts można zdefiniować interfejsy użytkownika oraz odpowiedzi API:

export interface User {
  id: string;
  name: string;
  email: string;
}

export interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

Więcej o organizacji typów w projekcie dowiesz się z lekcji #Organizacja typów w projekcie.

Typowanie hooków jest równie istotne. Przykład hooka useUser po migracji do TypeScript:

const useUser = (id: string) => {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetchUser(id)
      .then(user => setUser(user))
      .finally(() => setLoading(false));
  }, [id]);

  return { user, loading };
};

Więcej o typowaniu hooków dowiesz się z następnej sekcji.

Obsługa wyzwań technicznych

Podczas migracji mogą pojawić się różne wyzwania techniczne. Na przykład, problemy z defaultProps można rozwiązać za pomocą typu Partial. Rozważmy interfejs Props:

interface Props {
  value?: string;
  onChange: (value: string) => void;
}

const defaultProps: Partial<Props> = {
  value: ''
};

Innym wyzwaniem jest stosowanie generyków w Higher-Order Components (HOC). Przykładowa implementacja HOC z wykorzystaniem generyków:

function withLoader<P extends object>(
  WrappedComponent: React.ComponentType<P>
) {
  return function WithLoaderComponent(
    props: P & { loading?: boolean }
  ) {
    if (props.loading) {
      return <div>Loading...</div>;
    }
    return <WrappedComponent {...props} />;
  };
}

Więcej o typowaniu HOC dowiesz się z lekcji #Higher Order Components (HOC).

Typowanie ref w komponentach również wymaga uwagi. Przykład komponentu TextInput z typowanym refem:

const TextInput = React.forwardRef<
  HTMLInputElement,
  { placeholder: string }
>((props, ref) => (
  <input ref={ref} {...props} />
));

Więcej o typowaniu ref dowiesz się z lekcji #useRef.

Rekomendowane narzędzia i materiały

Aby ułatwić proces migracji, warto korzystać z odpowiednich narzędzi i materiałów, które wspomogą zarówno konfigurację środowiska, jak i samą migrację.

Rekomendowane narzędzia:

  • Edytor Cursor: Zapewnia wsparcie TypeScript i analizę błędów w czasie rzeczywistym, ułatwiając pisanie i debugowanie błędów. Jeżeli chcesz poznać możliwości Cursora, możesz to zrobić z naszym kursem “Cursor: Programuj z AI”.

  • ts-migrate: Narzędzie do automatycznej konwersji plików JavaScript do TypeScript, znacznie przyspieszające proces migracji.

  • type-coverage: Umożliwia śledzenie postępów migracji poprzez zbieranie i raportowanie błędów typów.

  • TypeScript ESLint: Pomaga w utrzymaniu spójności kodu poprzez analizę statyczną i wymuszanie standardów kodowania.

  • Prettier: Dba o jednolite formatowanie kodu, co zwiększa jego czytelność i ułatwia współpracę w zespole.

Rekomendowane materiały dodatkowe:

  • TypeScript Handbook - React: Oficjalna dokumentacja TypeScript z sekcją poświęconą integracji z React.

  • Sentry's TypeScript Migration Story: Studium przypadku firmy Sentry, które opisuje ich doświadczenia z migracją do TypeScript.

Podsumowanie

Migracja projektu React.js do TypeScript przynosi wiele korzyści, ale często stanowi duże wyzwanie. Skupienie się na systematycznym podejściu, wykorzystaniu odpowiednich narzędzi oraz przestrzeganiu najlepszych praktyk jest bardzo ważne dla sukcesu inicjatywy.

Dzięki temu proces migracji stanie się mniej uciążliwy, a zespół programistów będzie mógł cieszyć się zaletami typowania statycznego, takimi jak lepsza czytelność kodu i wcześniejsze wykrywanie błędów.
