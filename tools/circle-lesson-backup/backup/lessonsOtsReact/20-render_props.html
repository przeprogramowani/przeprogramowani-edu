W tej lekcji skupimy się na wykorzystaniu TypeScript wraz ze wzorcem Render Props.

Render Props to technika, która polega na przekazywaniu funkcji jako props do komponentu w celu dynamicznego renderowania zawartości. Zamiast komponentu samodzielnie decydować, co ma zostać wyrenderowane, przekazuje kontrolę nad renderowaniem do komponentu nadrzędnego.

Zalety stosowania Render Props

  • Reużywalność: Umożliwia tworzenie komponentów, które mogą być wykorzystywane w różnych kontekstach bez modyfikacji ich logiki wewnętrznej.

  • Inwersja kontroli: Komponent nadrzędny kontroluje sposób renderowania zawartości, co zwiększa elastyczność.

Podstawowa implementacja Render Props

Zacznijmy od stworzenia generycznego komponentu, który może renderować dowolne elementy za pomocą Render Props.

// List.tsx
import React from "react";

type Props<Item> = {
  items: Item[];
  renderItem: (item: Item) => React.ReactNode;
};

export function List<Item>({ items, renderItem }: Props<Item>) {
  return <ul>{items.map(renderItem)}</ul>;
}

Wyjaśnienie typowania:

  • Props<Item>: Generyczny typ props, gdzie Item reprezentuje typ pojedynczego elementu listy.

  • items: Item[]: Tablica elementów do wyrenderowania.

  • renderItem: (item: Item) => React.ReactNode: Funkcja renderująca pojedynczy element.

// App.tsx
import React from "react";
import { List } from "./List";
import { Pokemon } from "./api/pokemon";

function App() {
  const pokemons: Pokemon[] = getPokemons(); // Funkcja zwracająca tablicę Pokemonów

  return (
    <List
      items={pokemons}
      renderItem={(pokemon) => (
        <li key={pokemon.name}>
          <a href={pokemon.url} target="_blank" rel="noreferrer">
            {pokemon.name}
          </a>
        </li>
      )}
    />
  );
}

export default App;

Zastosowanie Render Props w oparciu o generyki daje nam następujących korzyści:

  • Bezpieczeństwo typów: TypeScript zapewnia, że renderItem otrzymuje odpowiedni typ Item.

  • Reużywalność: Komponent może być używany z dowolnym typem danych, np. Pokemon, Trainer, itp.

Wykorzystanie as prop dla większej elastyczności

Aby zwiększyć elastyczność naszego komponentu <List />, dodamy opcjonalny prop as, który pozwoli na zmianę typu HTML-owego elementu otaczającego listę.

// List.tsx
import React from "react";

interface BaseListProps<Item, As extends React.ElementType> {
  items: Item[];
  renderItem: (item: Item) => React.ReactNode;
  as?: As;
}

type ListProps<Item, As extends React.ElementType> = BaseListProps<Item, As> &
  Omit<React.ComponentPropsWithoutRef<As>, keyof BaseListProps<Item, As>>;

export function List<Item, As extends React.ElementType = "ul">({
  items,
  renderItem,
  as,
  ...rest
}: ListProps<Item, As>) {
  const Component = as || "ul";
  return <Component {...rest}>{items.map(renderItem)}</Component>;
}

Wyjaśnienie typowania:

  • As extends React.ElementType: Typ generyczny As musi być typem elementu React (np. 'div', 'ul', React.Fragment).

  • as?: As: Opcjonalny prop określający typ HTML-owego elementu otaczającego.

  • Omit<React.ComponentPropsWithoutRef<As>, keyof Props<Item, As>>: Usuwa z propsów <List /> te, które są już zdefiniowane w Props, aby uniknąć konfliktów.

// App.tsx
import React from "react";
import { List } from "./List";
import { Pokemon } from "./api/pokemon";

function App() {
  const pokemons: Pokemon[] = getPokemons();

  return (
    <List
      as="div"
      items={pokemons}
      renderItem={(pokemon) => (
        <div key={pokemon.name}>
          <a href={pokemon.url} target="_blank" rel="noreferrer">
            {pokemon.name}
          </a>
        </div>
      )}
      className="pokemon-list"
    />
  );
}

export default App;

Korzyści ze stosowania Render Props:

  • Elastyczność: Możemy zmienić element otaczający bez modyfikacji komponentu <List />.

  • Kompatybilność: Props specyficzne dla danego elementu (np. className, id) są automatycznie dostępne dzięki użyciu Omit.

Najlepsze praktyki

  1. Korzystaj z generyków: Umożliwia tworzenie elastycznych i wielokrotnego użytku komponentów.

  2. Używaj Omit i Partial: Pomagają w zarządzaniu propsami, eliminując konflikty i umożliwiając opcjonalne propsy.

  3. Unikaj nadmiernej złożoności: Wzorzec Render Props oferuje duże możliwości, ale może skomplikować strukturę komponentów. Rozważ alternatywy, takie jak hooki, gdy jest to możliwe.

  4. Zachowaj spójność API: Jeśli używasz Render Props, trzymaj się jednego sposobu przekazywania funkcji renderującej (np. zawsze przez children lub przez dedykowany prop jak renderItem).
