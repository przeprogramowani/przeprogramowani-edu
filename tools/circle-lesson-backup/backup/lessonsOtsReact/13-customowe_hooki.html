Tworzenie własnych hooków w React pozwala na reużywanie logiki komponentów, co prowadzi do czystszego i bardziej zorganizowanego kodu. Integracja TypeScript z customowymi hookami dodatkowo zwiększa bezpieczeństwo typów i czytelność kodu.

W tej lekcji omówimy zaawansowane techniki budowania customowych hooków w React z wykorzystaniem TypeScript:

Użycie as const do poprawnego wnioskowania typu tuple

Operator as const w TypeScript pozwala dokładnie typowanie zwracanych wartości, co jest szczególnie przydatne przy zwracaniu API customowego hooka. Dzięki temu TypeScript traktuje zwrócony array jako tuple o stałych typach elementów:

import { useState, useCallback } from 'react';

export const useToggle = (initial: boolean = false) => {
  const [state, setState] = useState<boolean>(initial);

  const toggle = useCallback(() => setState(prev => !prev), []);

  return [state, toggle] as const;
};

Dzięki temu możemy skorzystać z useToggle w komponencie mając szczegółowe informacje o typach:

import React from 'react';
import { useToggle } from './useToggle';

const ToggleComponent = () => {
  const [isToggled, toggle] = useToggle();

  return (
    <div>
      <p>Status: {isToggled ? 'Włączony' : 'Wyłączony'}</p>
      <button onClick={toggle}>Przełącz</button>
    </div>
  );
};

Dzięki temu isToggled ma typ boolean, a toggle ma typ () => void. Bez as const obydwie zmienne miałyby typ boolean | () => void.

Dlaczego warto stosować as const zwracając tuple?

  • Stałe typowanie: as const zapewnia, że TypeScript traktuje zwrócony array jako tuple o określonych typach.

  • Bezpieczeństwo typów: Zapobiega błędom wynikającym z nieoczekiwanych zmian w strukturze zwracanych danych.

Zarządzanie złożonym stanem za pomocą unii

Zarządzanie złożonym stanem w aplikacjach React często wymaga reprezentowania różnych stanów, takich jak loading, success czy error. TypeScript umożliwia wykorzystanie typów unii, aby precyzyjnie określić możliwe wartości stanu, co poprawia autouzupełnianie i zapobiega błędom:

import { useState, useEffect } from 'react';

type FetchState<T> = 
  | { status: 'loading' }
  | { status: 'success'; data: T }
  | { status: 'error'; error: Error };

export const useFetchData = <T>(url: string) => {
  const [state, setState] = useState<FetchState<T>>({ status: 'loading' });

  useEffect(() => {
    let isMounted = true;
    fetch(url)
      .then(async response => {
        if (!response.ok) throw new Error('Network response was not ok');
        const data: T = await response.json();
        if (isMounted) setState({ status: 'success', data });
      })
      .catch(error => {
        if (isMounted) setState({ status: 'error', error });
      });
    return () => {
      isMounted = false;
    };
  }, [url]);

  return state;
};

Taki customowy hook możemy użyć w następujący sposób w komponencie:

import React from 'react';
import { useFetchData } from './useFetchData';

interface User {
  id: number;
  name: string;
  email: string;
}

const UserProfile = ({userId}: {userId: number}) => {
  const fetchState = useFetchData<User>(`https://api.example.com/users/${userId}`);

  if (fetchState.status === "loading") return <p>Ładowanie...</p>;
  if (fetchState.status === "error") return <p>Błąd: {fetchState.error.message}</p>;

  return (
    <div>
      <h1>{fetchState.data.name}</h1>
      <p>Email: {fetchState.data.email}</p>
    </div>
  );
};

Dlaczego warto korzystać z unii?

  • Autouzupełnianie: TypeScript zapewnia autouzupełnianie dla możliwych stanów.

  • Bezpieczeństwo typów: Unia typów zapobiega przypisaniu nieistniejących stanów: np. dostęp do data w stanie loading.


Stosowanie discrimnated tuples dla złożonych wartości zwrotnych

Przedstawione powyżej podejście możemy popchnąć dalej stosując tzw. Discriminated tuples. Pozwalają one na precyzyjne powiązanie różnych typów danych w tuple, co zwiększa bezpieczeństwo i czytelność kodu, szczególnie podczas destrukturyzacji zwracanych wartości z hooków.

import { useState, useEffect } from 'react';

type AsyncResult<T> = 
  | ['idle', undefined]
  | ['pending', undefined]
  | ['success', T]
  | ['failure', Error];

export const useAsync = <T>(asyncFunction: () => Promise<T>, deps: any[] = []): AsyncResult<T> => {
  const [result, setResult] = useState<AsyncResult<T>>(['idle', undefined]);

  useEffect(() => {
    setResult(['pending', undefined]);

    asyncFunction()
      .then(data => {
        setResult(['success', data]);
      })
      .catch(error => {
        setResult(['failure', error]);
      });
  }, deps);

  return result;
};

useAsync możemy użyć w następujący sposób w komponencie:

import React from 'react';
import { useAsync } from './useAsync';

interface Post {
  id: number;
  title: string;
  content: string;
}

const PostList = () => {
  const [status, data, error] = useAsync<Post[]>(async () => {
    const response = await fetch('https://api.example.com/posts');
    if (!response.ok) throw new Error('Failed to fetch posts');
    return response.json();
  }, []);

  if (status === 'idle') return <p>Wprowadź zapytanie.</p>;
  if (status === 'pending') return <p>Ładowanie postów...</p>;
  if (status === 'failure') return <p>Błąd: {error.message}</p>;
  return (
    <ul>
      {data.map(post => (
        <li key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.content}</p>
        </li>
      ))}
    </ul>
  );
};

Dlaczego warto korzystać z discriminated tuples?

  • Precyzyjne typowanie: Każdy stan ma dokładnie określony typ.

  • Czytelność: Jasne powiązanie między statusem a danymi lub błędami.


Lepsze wrappery useState z wykorzystaniem function overload

Przeciążania funkcji (ang. function overload) pozwalają na definiowanie różnych sygnatur funkcji, co jest szczególnie przydatne przy opakowywaniu hooków takich jak useState. Dzięki temu customowy hook może zachować pełną dokładność typów charakterystyczną dla oryginalnego hooka, która uwzględnia różne przypadki użycia.

Jest to technika stosowana w typach Reacta dla wbudowanych hooków, który możemy przenieść na grunt customowych rozwiązań:

export function usePersistedState<T>(key: string, initial: T): [T, Dispatch<SetStateAction<T>>];
export function usePersistedState<T>(key: string): [T | undefined, Dispatch<SetStateAction<T | undefined>>];
export function usePersistedState<T>(key: string, initial?: T) {
  // Pobranie początkowej wartości z localStorage
  const getInitialValue = (): T | undefined => {
    const storedValue = localStorage.getItem(key);
    if (storedValue !== null) {
      return JSON.parse(storedValue) as T;
    }
    return initial;
  };

  const [state, setState] = useState<T | undefined>(getInitialValue);

  // Efekt zapisujący wartość do localStorage przy zmianie stanu
  useEffect(() => {
    if (state !== undefined) {
      localStorage.setItem(key, JSON.stringify(state));
    }
  }, [key, state]);

  return [state, setState] as const;
}

Hooka w rodzaju usePersistedState możemy użyć w komponencie w następujący sposób:

import React from 'react';
import { usePersistedState } from './usePersistedState';

const Settings = () => {
  const [theme, setTheme] = usePersistedState<string>('theme', 'light');
  const [volume, setVolume] = usePersistedState<number>('volume');

  return (
    <div>
      <h1>Settings</h1>
      <div>
        <p>Current Theme: {theme}</p>
        <button onClick={() => setTheme('light')}>Light</button>
        <button onClick={() => setTheme('dark')}>Dark</button>
      </div>

      <div>
        <p>Volume: {volume}</p>
        <input
          type="range"
          min="0"
          max="100"
          value={volume}
          onChange={(e) => setVolume(Number(e.target.value))}
        />
      </div>
    </div>
  );
};

export default Settings;

Dlaczego warto korzystać z function overloads?

  • Elastyczność: Customowy hook jest odpowiednio typowany zarówno dla inicjalizację ze stanem, jak i bez.

  • Pełna funkcjonalność i bezpieczeństwo: Zachowuje oryginalne przeciążenia useState, umożliwiając bezpieczną obsługę różnych scenariuszy.
