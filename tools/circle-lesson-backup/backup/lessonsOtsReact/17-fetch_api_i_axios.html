Wprowadzenie

Nowoczesne aplikacje frontendowe działają w złożonych i wielowarstwowych środowiskach, w których komunikacja z backendem odgrywa kluczową rolę. Pobieranie, synchronizacja i aktualizacja danych muszą być realizowane w sposób skalowalny, przewidywalny i bezpieczny, aby zapewnić użytkownikowi płynne doświadczenie. Wybór odpowiedniej strategii oraz narzędzi ma bezpośredni wpływ na wydajność aplikacji, jej architekturę oraz łatwość utrzymania kodu.

Efektywna komunikacja z backendem to jednak nie tylko kwestia samego przesyłania danych – to także zarządzanie stanem, obsługa błędów, typowanie odpowiedzi i minimalizowanie potencjalnych niespójności. W tych obszarach TypeScript daje programistom narzędzia do precyzyjnego modelowania danych, a biblioteki takie jak axios, swr czy React Query wspierają ich w dostarczeniu docelowego rozwiązania.

Narzędzia do komunikacji z backendem

W naszym przeglądzie popularnych narzędzi skupimy się na kilku rozwiązaniach:

  • natywnym Fetch API,

  • uniwersalnej bibliotece Axios,

  • dopasowanej do Reacta bibliotece SWR,

  • a także bibliotece React Query, która zajmuje się również stanem globalnym.

Ocenimy ich cechy nie tylko w kontekście surowych możliwości, ale przede wszystkim pod kątem integracji w projektach wykorzystujących język TypeScript i zaawansowany system typów.

W tej lekcji rozpoczniemy od stosunkowo prostych narzędzi jak Fetch i Axios, a w kolejnej wprowadzimy skrojone na miarę rozwiązania z ekosystemu Reacta (SWR i React Query).

Fetch API

Fetch API to natywny mechanizm do wykonywania żądań HTTP z poziomu przeglądarki. Jest lekki, uniwersalny i nie wymaga instalowania dodatkowych bibliotek. Umożliwia wysyłanie zapytań do backendu w prosty sposób, wykorzystujące Promise jako format obsługi asynchroniczności.

Zalety

  • Rozwiązanie natywne - brak konieczności instalowania dodatkowych zależności.

  • Kontrola nad zawartością zapytań (np. nagłówki, body, metody HTTP).

  • Wspólne API dla backendu (node-fetch)

  • Ponowne wykorzystanie poznanego narzędzia w przypadku zmiany stacku technologicznego

Wady

  • Automatyczna obsługa błędów i ponawianie zapytań wymagają ręcznej implementacji.

  • Wymaga dodatkowego kodu do parsowania odpowiedzi w różnych formatach (np. response.json()).

  • Obsługa błędów jest mało intuicyjna – trzeba ręcznie sprawdzać status odpowiedzi.

  • Brak wbudowanego mechanizmu zarządzania cachem.

  • Budowane w podejściu JavaScript-first (czego konsekwencją jest chociażby brak wsparcia dla typów generycznych).

Axios (https://axios-http.com/)

Axios to popularna biblioteka do wykonywania żądań HTTP, zapewniająca bardziej rozbudowane i (naszym zdaniem) bardziej intuicyjne API niż Fetch. W kontekście projektów TypeScript, ogromną zaletą Axiosa są generyczne metody wpływające na typy odpowiedzi, a także wsparcie dla takich aspektów komunikacji z backendem jak opóźnienia, przechwytywanie zapytań czy procesowanie formatu JSON. Zdaniem wielu programistów, Axios jest optymalnym kompromisem między możliwościami a łatwością obsługi.

Zalety

  • Wykorzystywanie potencjału TypeScriptu – możliwość dodania generyków do typowania odpowiedzi.

  • Intuicyjne wsparcie dla obsługi błędów – żądania o statusie innym niż 2xx automatycznie rzucają wyjątek (w przypadku Fetch API należy polegać na dodatkowym polu response.ok).

  • Lepsze API do obsługi odpowiedzi i automatyczne parsowanie formatu JSON.

  • Obsługa interceptorów – możliwość wprowadzenia globalnej logiki dla zapytań lub odpowiedzi (np. tokenów autoryzacyjnych).

  • Wsparcie dla bardziej zaawansowanych scenariuszy (retry, timeout).

  • Brak zależności na określony framework frontendowy

Wady

  • Brak wbudowanego systemu cache'owania.

  • Wymaga niezależnego rozwiązania do zarządzania stanem aplikacji

Podsumowanie

Poznane w tej lekcji Fetch API i Axios to obecnie najbardziej uniwersalne rozwiązania do pobierania danych w sposób asynchroniczny. Jak łatwo zauważyć, chociaż ich obsługa jest stosunkowo prosta, to nie są to rozwiązania tworzone z myślą o aplikacjach Reactowych.

Z tego powodu, w kolejnej lekcji poznasz dwa bardziej zaawansowane rozwiązania, które obudowują Fetch API / Axiosa dodatkowymi możliwościami i strategiami pobierania danych.
