Compound Components to wzorzec projektowy w React, który umożliwia tworzenie elastycznych i złożonych interfejsów użytkownika poprzez komponowanie mniejszych, współpracujących ze sobą komponentów. Zamiast przekazywać wiele propsów do pojedynczego komponentu, dzielimy funkcjonalność na mniejsze, bardziej specjalizowane komponenty, które współdzielą stan i logikę poprzez kontekst.

Zalety:

  • Modularność: Łatwiejsze zarządzanie i ponowne użycie.

  • Elastyczność: Użytkownicy mogą komponować komponenty zgodnie z potrzebami.

  • Czystość API: Mniej propsów do przekazywania w jednym komponencie, bardziej czytelny kod.

<Tabs>
    <Tabs.Title>Admin</Tabs.Title>
    <Tabs.Item id="overview">Overview</Tabs.Item>
    <Tabs.Item id="settings">Settings</Tabs.Item>
</Tabs>

Podstawowa implementacja

Zaczynamy od zdefiniowania typów dla głównego komponentu (Tabs) oraz jego podkomponentów (Tabs.Title i Tabs.Item).

// Typy dla głównego komponentu Tabs
interface TabsProps {
    children: React.ReactNode;
}

// Typy dla Tabs.Title
interface TitleProps {
    children: React.ReactNode;
}

// Typy dla Tabs.Item
interface ItemProps {
    id: string;
    isActive?: boolean;
    onClick?: () => void;
    children: React.ReactNode;
}

// Interfejs komponentu głownego, który wymusi zdefiniowanie podkomponentów
interface TabsComponent {
  (props: TabsProps): JSX.Element;
  Title: React.ComponentType<TitleProps>;
  Item: React.ComponentType<ItemProps>;
}

Następnie definiujemy główny komponent Tabs jako funkcjonalny komponent, który może zawierać dodatkowe właściwości (Title i Item).

// Główny komponent Tabs
const Tabs: TabsComponent = ({children, ...props}) => {
  const [activeTab, setActiveTab] = useState<string>();

  return (
    <div className='tabs'>
      {Children.map(children, (child) => {
        const item = child as ReactElement<ItemProps>;

        if (item.type === Tabs.Item) {
          const isActive = item.props.id === activeTab;
          const onClick = () => {
            setActiveTab(item.props.id);
            item.props.onClick?.();
          };
          return cloneElement(item, {isActive, onClick});
        } else {
          return child;
        }
      })}
    </div>
  );
};

Zostało nam zdefiniowanie komponentów podrzędnych Title i Item. Dzięki typowi TabsComponent dopóki tego nie zrobimy Tabs będzie zwracał błąd kompilacji - podkomponenty są częścią jego interfejsu.

// Definiowanie podkomponentów
const Title = ({children}: TitleProps) => (
  <div className='tabs-title'>{children}</div>
);

const Item = ({children, isActive, onClick}: ItemProps) => (
  <div className={`tabs-item ${isActive ? "active" : ""}`} onClick={onClick}>
    {children}
  </div>
);

// Przypisywanie podkomponentów do głównego komponentu
Tabs.Title = Title;
Tabs.Item = Item;

export {Tabs};

Szczegółowe omówienie React.Children.map i cloneElement

Skupmy się na najbardziej nieoczywistym elemencie implementacji wzorca Compound Components:

{Children.map(children, (child) => {
        const item = child as ReactElement<ItemProps>;

        if (item.type === Tabs.Item) {
          const isActive = item.props.id === activeTab;
          const onClick = () => {
            setActiveTab(item.props.id);
            item.props.onClick?.();
          };
          return cloneElement(item, {isActive, onClick});
        } else {
          return child;
        }
})}

React.Children.map pozwala na iterację po dzieciach komponentu, niezależnie od tego, czy są one pojedynczym elementem, tablicą czy innym typem.

React.cloneElement umożliwia klonowanie elementu React i przekazywanie do niego dodatkowych propsów.

Aby zapewnić poprawne typowanie dzieci, musimy skorzystać z asercji typu na ReactElement przekazując ItemProps jako generyka

const element = child as ReactElement<ItemProps>;

Następnie weryfikujemy czy dany element jest faktycznie TabsItem za pomocą właściwości item.type. Jeżeli tak, to dodajemy do niego propsy isActive oraz onClick za pomocą cloneElement. Dzięki TS mamy pewność, że przekazujemy propsy zgodnie z interfejsem TabsItemProps.

Compound Components z forwardRef

W kontekście Compound Components, gdzie mamy zestaw powiązanych komponentów współdzielących stan lub logikę, forwardRef odgrywa kluczową rolę z kilku powodów:

  1. Bezpośredni dostęp do elementów DOM - umożliwia rodzicowi lub zewnętrznym komponentom bezpośredni dostęp do elementów DOM wewnątrz podkomponentów. Przykładowo, może to być potrzebne do ustawiania fokusu na konkretnym elemencie, mierzenia jego rozmiaru czy manipulowania przewijaniem.

  2. Integracja z bibliotekami zewnętrznymi - czasami konieczne jest zintegrowanie komponentów z zewnętrznymi bibliotekami, które wymagają bezpośredniego dostępu do elementów DOM. forwardRef umożliwia przekazywanie ref do odpowiedniego elementu wewnątrz podkomponentu.

  3. Zachowanie elastyczności API - dzięki forwardRef, API komponentów staje się bardziej elastyczne i przejrzyste, umożliwiając użytkownikom komponentów bardziej zaawansowane manipulacje bez konieczności modyfikowania struktury komponentów.

Rozważmy przykład z wcześniejszego komponentu Tabs, gdzie podkomponenty Tabs.Title i Tabs.Item mogą potrzebować dostępu do elementów DOM.

export interface TabsItemProps {
  id: string;
  isActive?: boolean;
  onClick?: () => void;
  children: React.ReactNode;
}

export const TabsItem = forwardRef<HTMLDivElement, TabsItemProps>(
  ({children, isActive = false, onClick}, ref) => (
    <div
      ref={ref}
      className={`${styles.item} ${isActive ? styles.active : ""}`}
      onClick={onClick}
    >
      {children}
    </div>
  )
);

TabsItem.displayName = "TabsItem";

Przy stosowaniu forwardRef musimy zmienić definicję TabsComponent, aby TypeScript brał pod uwagę refa w interfejsie komponentu:

interface TabsComponent {
  (props: TabsProps): JSX.Element;
  Title: React.ComponentType<TitleProps>;
  Item: typeof TabsItem;
}

Teraz możemy skorzystać z refa w następujący sposób:

const SimpleTabsApp = () => {
    const overviewRef = useRef<HTMLDivElement>(null); // TypeScript wskaże nam jakiego HTMLElement oczekuje ref w Tabs.Item

    const handleClick = () => {
        if (overviewRef.current) {
            overviewRef.current.scrollIntoView({ behavior: "smooth" });
        }
    };

    return (
        <Tabs>
            <Tabs.Title>Admin</Tabs.Title>
            <Tabs.Item id="overview" ref={overviewRef} onClick={handleClick}>
                Overview
            </Tabs.Item>
            <Tabs.Item id="settings">Settings</Tabs.Item>
        </Tabs>
    );
};

export default TabsDemo;
