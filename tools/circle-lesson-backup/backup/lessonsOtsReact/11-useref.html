W tej lekcji skupimy się na dwóch mechanizmach, które są nieodłączną częścią pracy z referencjami w React: useRef i forwardRef.

Oba te mechanizmy są szczególnie przydatne w sytuacjach, gdy musimy manipulować bezpośrednio DOM-em lub przekazywać referencje do zagnieżdżonych komponentów.

Omówimy zarówno podstawy ich użycia, jak i zaawansowane techniki, takie jak typowanie referencji w TypeScript oraz radzenie sobie z wyzwaniami związanymi z komponentami generycznymi.

Typowanie referencji za pomocą useRef

useRef jest hookiem, który pozwala na przechowywanie referencji do elementów DOM lub innych zmiennych, które mają pozostać niezmienne przez cały cykl życia komponentu. Jest to użyteczne, gdy potrzebujemy uzyskać dostęp do elementu w DOM lub przechować jakąś wartość, dla których nie chcemy, by powodowały ponowne renderowanie komponentu.

Przykład użycia useRef do uzyskania dostępu do elementu DOM:

import React, { useRef } from 'react';

const TextInputWithFocusButton = () => {
  // Tworzymy ref dla elementu input
  const inputRef = useRef<HTMLInputElement>(null);

  const focusInputOnClick = () => {
    // Odwołanie do ref i ustawienie focusa na elemencie
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <>
      <input ref={inputRef} type="text" />
      <button onClick={focusInputOnClick}>Focus the input</button>
    </>
  );
};

export default TextInputWithFocusButton;

W tym przykładzie inputRef jest referencją do elementu input, a dzięki inputEl.current uzyskujemy dostęp do tego elementu, aby wywołać na nim metodę focus(). Ważne jest, aby pamiętać o sprawdzeniu, czy current nie jest null, ponieważ TypeScript nie zna dokładnego typu elementu, dopóki nie zostanie przypisany do referencji.

Typowanie useRef z ElementRef

Integrując Reacta z TSem musimy mieć pewność, że nasza referencja korzysta z odpowiedniego typu dla elementu DOM. Typów dla elementów DOM jest pełno i ciężko je wszystkie zapamiętać. Na szczęście możemy skorzystać z generycznego typu pomocniczego ElementRef, który automatycznie typuje naszą referencję do odpowiedniego typu HTML na podstawie stringa z nazwą elementu.

import React, { useRef, ElementRef } from 'react';

const HelloDialog = () => {
  const dialogRef = useRef<ElementRef<'dialog'>>(null);

  return <dialog ref={dialogRef}>Hello</dialog>;
};

W powyższym przykładzie używamy ElementRef, który automatycznie wyciąga odpowiedni typ z elementu, w tym przypadku HTMLDialogElement.

forwardRef - przekazywanie referencji do komponentów

forwardRef to funkcja wyższego rzędu, która umożliwia przekazywanie referencji do komponentów funkcyjnych. Dzięki forwardRef możemy przekazać referencję z komponentu nadrzędnego do komponentu podrzędnego, co jest szczególnie przydatne w przypadku komponentów kontenerowych, które zarządzają stanem swoich dzieci.

import React, { forwardRef } from 'react';

const InputWithFocusButton = forwardRef<ElementRef<'input'>, {}>((props, ref) => (
  <input ref={ref} type="text" />
));

const ParentComponent = () => {
  const inputRef = useRef<ElementRef<'input'>>(null);

  const focusInput = () => {
    inputRef.current?.focus();
  };

  return (
    <>
      <InputWithFocusButton ref={inputRef} />
      <button onClick={focusInput}>Focus Input</button>
    </>
  );
};

W powyższym przykładzie forwardRef jest użyte do przekazywania referencji do wewnętrznego elementu input w komponencie InputWithFocusButton.

Problemy z inferencją typu w forwardRef dla komponentów generycznych

Jednak w przypadku komponentów generycznych (generic components), takich jak tabelki czy listy, TypeScript może mieć problemy z poprawnym rozpoznaniem typów w odniesieniu do przekazywanych referencji.

import React, { forwardRef, useRef } from 'react';

type TableProps<T> = {
  data: T[];
  renderRow: (row: T) => React.ReactNode;
};

const Table = <T,>({ data, renderRow }: TableProps<T>, ref: React.Ref<HTMLTableElement>) => (
  <table ref={ref}>
    <tbody>
      {data.map((row, index) => (
        <tr key={index}>{renderRow(row)}</tr>
      ))}
    </tbody>
  </table>
);

const ForwardRefTable = forwardRef(Table);

const App = () => {
  const tableRef = useRef<HTMLTableElement>(null);

  return (
    <ForwardRefTable
      ref={tableRef}
      data={['Row 1', 'Row 2']}
      renderRow={(row) => <td>{row}</td>} // Type 'unknown' is not assignable to type 'ReactNode'.ts(2322)
    />
  );
};

W powyższym kodzie TypeScript nie potrafi poprawnie wywnioskować typu danych w renderRow. Zamiast oczekiwanego typu string, w wielu przypadkach typ row zostanie rozpoznany jako unknown. Dzieje się tak, ponieważ definicja typu React.forwardRef nie wspiera dobrze inferencji typów generycznych.

Jak działa FixedForwardRef?

FixedForwardRef to customowa funkcja, która zastępuje standardowe typowanie forwardRef i poprawia inferencję typów w przypadku komponentów generycznych.

export function fixedForwardRef<T, P = {}>(
  render: (props: React.PropsWithoutRef<P>, ref: React.Ref<T>) => React.ReactNode
): (props: P & React.RefAttributes<T>) => React.ReactNode {
  return React.forwardRef(render) as any;
}

  1. Przyjmuje funkcję renderującą, która działa jak standardowy React.forwardRef, ale typy T (typ referencji) i P (typ propsów) są jawnie przekazywane.

  2. Zwraca nową funkcję, która dodaje RefAttributes<T> do przekazywanych propsów, dzięki czemu TypeScript może poprawnie wnioskować typ referencji.

  3. Używa as any, aby obejść ograniczenia standardowego typowania forwardRef (ten fragment jest bezpieczny w tym kontekście, ponieważ typowanie jest precyzyjnie kontrolowane na poziomie funkcji).

Warto ją umieścić fixedForwardRef w głównym pliku z typami pomocniczymi Twojego projektu i wykorzystywać do obsługi komponentów generycznych wykorzystujących referencje.

Przykład zastosowania FixedForwardRef

Załóżmy, że mamy generyczny komponent formularza, który wyświetla różne pola i pozwala uzyskać referencję do całego formularza.

import React, {useRef} from "react";
import {fixedForwardRef} from "@types/utils"

type FormProps<T> = {
  data: T;
  renderFields: (data: T) => React.ReactNode;
};

const Form = <T,>(
  props: FormProps<T>,
  ref: React.ForwardedRef<HTMLFormElement>
) => {
  return (
    <form ref={ref}>
      {props.renderFields(props.data)}
      <button type='submit'>Submit</button>
    </form>
  );
};

const ForwardReffedForm = fixedForwardRef(Form);

const App = () => {
  const formRef = useRef<HTMLFormElement>(null);

  const handleSubmit = () => {
    if (formRef.current) {
      console.log("Form submitted!", new FormData(formRef.current));
    }
  };

  const formData = {
    name: "",
    age: "",
  };

  const renderFields = (data: typeof formData) => (
    <>
      <label>
        Name:
        <input name='name' defaultValue={data.name} />
      </label>
      <label>
        Age:
        <input name='age' defaultValue={data.age} type='number' />
      </label>
    </>
  );

  return (
    <>
      <ForwardReffedForm
        ref={formRef}
        data={formData}
        renderFields={renderFields}
      />
      <button onClick={handleSubmit}>Submit Form</button>
    </>
  );
};

export default App;

  • FixedForwardRef w akcji: Funkcja fixedForwardRef poprawia inferencję typów dla komponentu Form. Dzięki temu TypeScript prawidłowo rozpoznaje, że ref odnosi się do elementu HTMLFormElement, a data jest obiektem zawierającym pola formularza.

  • Dynamiczne pola: Funkcja renderFields generuje pola formularza na podstawie danych wejściowych, a ich typy są automatycznie wnioskowane.

  • Zarządzanie referencją: Referencja formRef jest używana do odczytu danych formularza przy użyciu FormData.

Zaawansowane wzorce pracy z useRef i forwardRef

Korzystanie z useImperativeHandle

Kiedy używamy forwardRef, możemy skorzystać z hooka useImperativeHandle, aby dostosować zachowanie referencji przekazywanych do komponentu. Dzięki temu możemy kontrolować, co dokładnie jest dostępne dla komponentu nadrzędnego.

import React, {forwardRef, useImperativeHandle, useRef, useState} from "react";

interface CountdownHandle {
  start: () => void;
  reset: () => void;
};

const Countdown = forwardRef<CountdownHandle>((props, ref) => {
  const intervalRef = useRef<number | null>(null);
  const [count, setCount] = useState(10);

  useImperativeHandle(ref, () => ({
    start() {
      if (!intervalRef.current) {
        intervalRef.current = setInterval(() => {
          setCount((prev) => (prev > 0 ? prev - 1 : 0));
        }, 1000);
      }
    },
    reset() {
      if (intervalRef.current) {
        clearInterval(intervalRef.current);
        intervalRef.current = null;
      }
      setCount(10);
    },
  }));

  return <div>Countdown: {count}</div>;
});

const App = () => {
  const countdownRef = useRef<CountdownHandle>(null);

  return (
    <>
      <Countdown ref={countdownRef} />
      <button onClick={() => countdownRef.current?.start()}>Start</button>
      <button onClick={() => countdownRef.current?.reset()}>Reset</button>
    </>
  );
};

export default App;

  • Hook useImperativeHandle pozwala na zadeklarowanie interfejsu, co będzie dostępne w ref.

  • Dzięki temu możemy przekazywać tylko te metody lub właściwości, które chcemy udostępnić komponentowi nadrzędnemu. W tym przypadku interfejs określa CountdownHandle.

  • Użycie tego wzorca pomaga w izolacji logiki komponentu dziecka względem rodzica.

Dynamiczne renderowanie różnych elementów DOM

W niektórych przypadkach komponenty muszą dynamicznie renderować różne elementy DOM w zależności od propsów lub innych czynników. Użycie typeof Component zamiast sztywnych typów takich jak ElementRef<"div"> zapewnia elastyczność i poprawność typowania nawet w dynamicznych scenariuszach.

Stworzymy komponent DynamicElement, który renderuje różne elementy DOM (div, button, input) w zależności od wartości propsów. Dzięki typeof możemy automatycznie typować referencję bez potrzeby ręcznej aktualizacji typów.

import React, {ElementRef, forwardRef, useRef} from "react";

type DynamicElementProps = {
  as: "div" | "button" | "input";
  label?: string;
};

const DynamicElement = forwardRef<HTMLElement, DynamicElementProps>(
  ({as, label, ...props}, ref) => {
    if (as === "button") {
      return (
        <button ref={ref as React.Ref<HTMLButtonElement>} {...props}>
          {label || "Button"}
        </button>
      );
    }
    if (as === "input") {
      return (
        <input
          ref={ref as React.Ref<HTMLInputElement>}
          {...props}
          placeholder={label}
        />
      );
    }

    return (
      <div ref={ref as React.Ref<HTMLDivElement>} {...props}>
        {label || "Div"}
      </div>
    );
  }
);
// Typ referencji bazujący na DynamicElement
type DynamicElementRef = ElementRef<typeof DynamicElement>;

const App = () => {
  const elementRef = useRef<DynamicElementRef>(null);

  const handleFocus = () => {
    if (elementRef.current instanceof HTMLInputElement) {
      elementRef.current.focus();
    }
  };

  const handleClick = () => {
    if (elementRef.current instanceof HTMLButtonElement) {
      alert("Button clicked!");
    }
  };

  return (
    <>
      <DynamicElement ref={elementRef} as='input' label='Type something' />
      <button onClick={handleFocus}>Focus Input</button>

      <DynamicElement ref={elementRef} as='button' label='Click me' />
      <button onClick={handleClick}>Trigger Button Action</button>

      <DynamicElement ref={elementRef} as='div' label='Just a Div' />
    </>
  );
};

export default App;

Dlaczego warto skorzystać z type DynamicElementRef = ElementRef<typeof DynamicElement>?

  • Typ ElementRef<typeof DynamicElement> automatycznie dostosowuje się do aktualnie renderowanego elementu (div, button, input), co pozwala na dynamiczne kontrolowanie referencji.

  • Elastyczność: Jeśli w przyszłości dodamy obsługę kolejnych elementów DOM, typ ElementRef<typeof DynamicElement> nie wymaga modyfikacji.

Najlepsze praktyki

  1. Stosuj dokładne typy elementów HTML
    Zawsze preferuj typowanie referencji z najbardziej szczegółowym typem, np. HTMLDivElement zamiast HTMLElement, aby uzyskać większą precyzję i uniknąć błędów w dostępie do metod i właściwości elementów. Jeżeli nie pamiętasz odpowiedniego typu, skorzystaj z ElementRef<“div”>.

  2. Używanie forwardRef z komponentami generycznymi
    W przypadku komponentów generycznych, takich jak tabela, musisz zadbać o odpowiednie zarządzanie referencjami. Możesz zastosować fixedForwardRef, aby obejść problemy z inferencją typu.

  3. Zarządzanie nullowymi referencjami w useRef
    Gdy używasz useRef z typami DOM, pamiętaj, aby sprawdzać, czy current nie jest null, szczególnie gdy elementy mogą być warunkowo renderowane lub mogą nie być dostępne przy pierwszym renderowaniu.

Podsumowanie

  • useRef jest uniwersalnym narzędziem do przechowywania referencji do DOM-u lub innych zmiennych, a dzięki typowaniu w TypeScript można znacznie poprawić czytelność i bezpieczeństwo kodu.

  • forwardRef to kluczowy mechanizm do przekazywania referencji między komponentami, który w połączeniu z useImperativeHandle pozwala na precyzyjne zarządzanie API komponentu.

  • Zrozumienie wyzwań z inferencją typów i zastosowanie takich rozwiązań jak fixedForwardRef pozwala unikać frustracji i problemów w bardziej zaawansowanych przypadkach.

Praktyka z omawianymi przykładami zapewni głębsze zrozumienie tych mechanizmów i pomoże Ci w codziennej pracy.
