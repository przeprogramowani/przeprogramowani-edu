Tworzymy prywatne repozytorium na CodeArtifact

Zaczniemy proces korzystania z prywatnego rejestru NPM od utworzenia repozytorium CodeArtifact w Konsoli AWS.

Publikujemy paczkę do CodeArtifact

Jak skorzystać z prywatnego repozytorium CodeArtifact w Twoim projekcie? W tym celu zapoznamy się z plikiem konfiguracyjnym .npmrc.

Szablon .npmrc dla rejestru AWS Code Artifact:

registry=https://<TWOJA-DOMENA-CA>-<TWOJE-AWS-ACCOUNT-ID>.d.codeartifact.<REGION-AWS>.amazonaws.com/npm/<TWOJE-REPOZYTORIUM-CA>/
//<TWOJA-DOMENA-CA>-<TWOJE-AWS-ACCOUNT-ID>.d.codeartifact.<REGION-AWS>.amazonaws.com/npm/<TWOJE-REPOZYTORIUM-CA>/:always-auth=true
//<TWOJA-DOMENA-CA>-<TWOJE-AWS-ACCOUNT-ID>.d.codeartifact.<REGION-AWS>.amazonaws.com/npm/<TWOJE-REPOZYTORIUM-CA>/:_authToken=<TWOJ-AUTH-TOKEN-CA>

Komenda AWS CLI do pobierania auth tokenu dla CA:

aws codeartifact get-authorization-token --domain <TWOJA-DOMENA-CA> --domain-owner <TWOJE-AWS-ACCOUNT-ID> --query authorizationToken --output text

Zarządzanie zależnościami

Zarządzanie zależnościami w projektach frontendowych, to kluczowe zadanie, które zapewnia bezpieczeństwo, wydajność i stabilność aplikacji. Wymaga to stałego monitorowania, aktualizacji i oceny zależności, aby zapobiegać problemom z bezpieczeństwem i niekompatybilnością. Oto kilka narzędzi i technik, które pomagają w tej pracy:

npm audit: Ta komenda analizuje zależności projektu i identyfikuje znane podatności. npm audit porównuje każdy pakiet z bazą danych podatności i dostarcza raport, który wskazuje konkretne problemy bezpieczeństwa wraz z zalecanymi aktualizacjami. Aby uruchomić to narzędzie, wystarczy wpisać npm audit w roocie projektu.

W przypadku zagrożeń oznaczonych jako critical warto podjąć natychmiastowe działania aktualizacji/naprawy problematycznego pakietu, niezależnie czy mamy do czynienia z dependency czy devDependency. Następnie koncentruj się na aktualizowaniu/naprawianiu zależności, które są powiązane z dependency Twojego projektu i są oznaczone jako high. Warto mieć na uwadze, że jeżeli podatność znajduje się w devDependency lub w drzewie zależności takowego devDependency, to zagrożenie dla użytkownika jest niskie, ponieważ taki kod nie trafi do paczki wynikowej - warto tutaj również dbać o rozwiązywanie podatności high, ale nie ma to aż tak wysokiego priorytetu.

npm outdated: Komenda ta sprawdza, które z zainstalowanych pakietów mają dostępne nowsze wersje. Pokazuje aktualną wersję, oczekiwaną wersję (zależną od zawartości package.json) oraz najnowszą dostępną wersję.

Proces aktualizacji zależności można zautomatyzować i tutaj świetnym rozwiązaniem jest Dependabot.

Dependabot dba o aktualność i bezpieczeństwo dependencji poprzez automatyczne wystawianie PRów do repozytorium. W 2019 został zakupiony przez GitHub, dzięki czemu otrzymaliśmy topową integrację z tym środowiskiem.

Rozwiązanie jest łatwe w konfiguracji, wystarczy utworzyć plik .github/dependabot.yml i wskazać nasze preferencje względem analizowanych folderów, częstotliwośći wystawiania PRów oraz ich ilości.

Jeżeli chodzi o częstotliwość aktualizacji depsów to moja rekomendacja jest następująca:

  1. Zależności minor/patch:

    1. Optymalnie: co tydzień

    2. Minimalnie: co dwa tygodnie

  2. Zależności major:

    1. Optymalnie: co miesiąc

    2. Minimimalnie: co kwartał

To co rozumiem przez “optymalnie/minimalnie” bierze pod uwagę, że żyjemy w prawdziwym świecie, w którym mamy wiele pilnych zadań walczących o naszą uwagę. Takie podejście dobrze sprawdzało się w realizowanych przeze mnie projektach, ale pamiętaj że decyzja o częstotliwości aktualizacji zawsze spoczywa na zespole i powinna być dostosowana do kontekstu firmy i wymagań projektu.


🧑🏻‍💻 Ćwiczenia praktyczne

  1. Utwórz prywatne repozytorium Code Artifact

    1. Skonfiguruj repozytorium w AWS Console

    2. Utwórz własny plik .npmrc zgodnie z szablonem

    3. Opublikuj paczkę @twoj-username-npm/design-tokens za pomocą npm publish

    4. Opublikuj paczkę za pomocą “Publish in AWS CodeArtifact”

      1. Pamiętaj o konfiguracji sekretów w ustawieniach Twojego repo/forka

      2. Pamiętaj o aktualizacji uprawnień “Read & Write” dla Github Actions w ustawieniach Twojego repo/forka

    5. Dla chętnych: Rozbuduj workflow o automatyczne tworzenie Github Release za pomocą akcji gh-release

  2. Skonfiguruj Dependabota dla examples/module4/lesson2/design-tokens w swoim projekcie zgodnie z dokumentacją

    1. Aktywuj rozszerzenie w ustawieniach forka/repo

    2. Ustaw cotygodniowy update depsów w .github/dependabot.yml dla /examples/module4/lesson2/desing-tokens

  1. Dla chętnych: Postaw lokale repozytorium w Verdaccio

    1. Zainstaluj Verdaccio

    2. Dostosuj konfigurację npm do lokalnego rejestru

    3. Opublikuj paczkę i sprawdź czy pojawiła się w lokalnym rejestrze

    4. Pobierz paczkę z lokalnego rejestru do dowolnego projektu
