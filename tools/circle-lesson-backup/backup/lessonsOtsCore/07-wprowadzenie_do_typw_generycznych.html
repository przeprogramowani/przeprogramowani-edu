Wprowadzenie

W każdym nowoczesnym języku programowania pojawiają się różnorodne mechanizmy pozwalające na tworzenie elastycznego, bezpiecznego i łatwego w utrzymaniu kodu. Wśród nich szczególne miejsce zajmują typy generyczne. Możemy je znaleźć w takich językach jak Java, C#, C++, Swift czy Kotlin, oraz właśnie w TypeScript.

Dlaczego twórcy języków programowania decydują się na wdrażanie tej funkcjonalności?

W świecie typów, generyki dają nam podobne korzyści jak funkcje i komponenty w świecie logiki biznesowej i interfejsów użytkownika:

  • 📈 Funkcje: Umożliwiają wyodrębnianie powtarzającej się logiki do jednego miejsca w projekcie. Dzięki temu, zamiast pisać ten sam kod wielokrotnie, możemy go zorganizować w jednej funkcji, którą można wywoływać w różnych miejscach. To zwiększa czytelność kodu i ułatwia jego utrzymanie.

function storeDetails(details: PersonalDetails) {
  ...
}

storeDetails(user);
storeDetails(admin);

  • 🌃 Komponenty: Umożliwiają wyodrębnianie reużywalnych elementów interfejsu użytkownika do wspólnego elementu, który można później parametryzować. Komponenty mogą być stosowane wielokrotnie w różnych częściach aplikacji, co upraszcza zarządzanie i rozwój UI.

function CallToAction({ onClick, label }: { onClick: () => void; label: string }) {
  return <button onClick={onClick}>{label}</button>;
}

// OtherComponent
return (
  <CallToAction onClick={handleSubmit} label="Submit" />;
  <CallToAction onClick={handleCancel} label="Cancel" />;
)

  • 🛠️ Generyki: Umożliwiają tworzenie określonych struktur (funkcji, klas i interfejsów), które są niezależne od konkretnego typu danych, z którym pracują. Dzięki temu możemy zdefiniować spójną logikę, która działa z różnymi typami. Takie podejście minimalizuje duplikację kodu i zwiększa jego elastyczność.

type AllFieldsRequired<T> = ...

type RestrictiveDetails = AllFieldsRequired<PersonalDetails>;
type RestrictiveMetadata = AllFieldsRequired<LogMetadata>;

Przykładowa refaktoryzacja z użyciem typu generycznego została przedstawiona poniżej - w efekcie, zamiast dwóch funkcji różniących się detalem, mamy jeden uniwersalny sposób na logowanie tablic:

// Przed - dwie funkcje realizujące ten sam cel, wspierające dwa typy argumentów
function printStringArray(arr: string[]): void {
    arr.forEach(item => console.log(item));
}

function printNumberArray(arr: number[]): void {
    arr.forEach(item => console.log(item));
}

// Po - jedna funkcja do obsługi tablic bliżej nieokreślonego typu ("T")
function printArray<T>(arr: T[]): void {
    arr.forEach(item => console.log(item));
}

Dzięki wprowadzeniu funkcji z typem generycznym, operacja logowania elementów tablic może być zaimplementowana tylko raz (wcześniej każdy typ wymuszał tworzenie odrębnej funkcji).

Konwencja nazewnicza

Być może zastanawiasz się co oznacza typ T, który wprowadziliśmy jako alternatywę dla typów string i number - przecież w programowaniu nazewnictwo powinno być intuicyjne, a T to zupełne przeciwieństwo tej praktyki?

Konwencja nazywania typów generycznych pojedynczymi literami wywodzi się z tradycji matematycznej, gdzie często używa się pojedynczych liter do oznaczania zmiennych lub nieznanych wartości. W programowaniu przyjęła się ze względu na zwięzłość, abstrakcyjność i uniwersalność, co ułatwia czytelność kodu i podkreśla ogólny charakter typów generycznych.

Wprowadzając typ T, wprowadzamy pewien symbol zastępczy, którego znaczenie celowo zaciemniamy - w końcu generyki dają nam elastyczność i mniejsze przywiązanie do jednego ustalonego typu. Zdarzają się jednak sytuacje (o tym dalej), kiedy chcemy użyć bardziej znaczącej nazwy i na szczęście TypeScript nie nakłada w tej kwestii wymagań czy ograniczeń.

Generyczne elementy języka

Cztery główne obszary języka TypeScript, w których możemy stosować typy generyczne, to:

  • aliasy typów

  • funkcje

  • klasy

  • interfejsy

Zobacz jak typy generyczne wpływają na każdy z tych elementów.

Aliasy typów

Aliasy typów, definiowane poprzez słowo kluczowe type, umożliwiają tworzenie typów generycznych:

type TypeName<T1, T2, ...> = ...

Taka konstrukcja oznacza, że będziemy tworzyć nowy typ (TypeName) oparty o istniejące cechy dwóch innych typów - T1 i T2 (podobnie jak gdybyśmy pisali funkcję z dwoma argumentami).

Typ Binding może przykładowo służyć do wprowadzenia pary klucz-wartość o dwóch dynamicznie określanych parametrach K i V:

type Binding<K, V> = {
  key: K;
  value: V;
};

const stringToNumber: Binding<string, number> = {
  key: 'foo',
  value: 42,
};

const objectToBoolean: Binding<object, boolean> = {
  key: { foo: 'bar' },
  value: true,
};

Poprzez aliasy typów możemy tworzyć dowolnie złożone powiązania, takie jak chociażby “trójki” parametrów czy inne typy, które pasują do naszej domeny lub problemu, z którym pracujemy:

type Three<A, B, C> = {
  a: A;
  b: B;
  c: C;
};

const three: Three<string, number, boolean> = {
  a: 'hello',
  b: 42,
  c: true,
};

Zauważ, że zarówno w typie Binding jak i Three, można zaobserwować podobną filozofię działania jak w przypadku klasycznych funkcji. Mamy określoną logikę (odpowiedzialność typu) i przekazywane parametry (parametry typów generycznych).

Funkcje

W przypadku funkcji, typ generyczny definiujemy poprzez umieszczenie go w ostrych nawiasach, zaraz za nazwą funkcji:

function genericFn<T>() { ... }

W przypadku funkcji strzałkowych, typ umieszczamy przed listą parametrów:

const arrowFn = <T,>() => { ... }

👉 Dla ciekawskich: W przypadku funkcji strzałkowych znajdujących się w plikach .tsx, wymagany jest zapis <T,> - dzięki niemu kompilator TS rozpoznaje różnicę pomiędzy generykami a JSX.

Generyczne parametry i wartości zwracane

Korzystając z funkcji generycznych możemy zadecydować, czy typ generyczny:

  • a) będzie dotyczył typu argumentów - w sytuacji, kiedy chcemy stworzyć uniwersalną funkcjonalność działającą w obrębie różnych typów danych (np. iterowanie po tablicach, filtrowanie, etc.)

  • b) typu danych wyjściowych - w sytuacji, kiedy typ argumentu jest stały (np. adres URL), a typ wartości zwracanej może się zmieniać na żądanie (np. pobieranie określonego formatu danych z API)

  • c) zarówno typu parametrów jak i danych wyjściowych - połączenie obu powyższych przypadków i wprowadzenie spójności pomiędzy argumentami a danymi wyjściowymi

// Funkcja generyczna dla typu argumentów
function logEntries<T>(entries: T[]): void { ... }

// Funkcja generyczna dla typu zwracanego obiektu
function fetchData<T>(url: string): T { ... }

// Funkcja generyczna wiążąca typ parametrów i wartości zwracanej
function processData<T>(params: T[]): T[] { ... }

Wiele parametrów generycznych

Korzystając z generyków możemy wprowadzić kilka parametrów typów, które określą relację pomiędzy poszczególnymi fragmentami takiej funkcji:

function createGenericTransform<T, K>(transform: (input: T) => K) {
  return (value: T): K => transform(value);
}

// Przykład użycia:
const stringToNumber = createGenericTransform<string, number>((val) => parseInt(val, 10));
const result1 = stringToNumber("123"); // Zwraca: 123, typ: number

const numberToString = createGenericTransform<number, string>((val) => val.toString());
const result2 = numberToString(42); // Zwraca: "42", typ: string

const arrayToLength = createGenericTransform<number[], number>((arr) => arr.length);
const result3 = arrayToLength([1, 2, 3, 4]); // Zwraca: 4, typ: number

Funkcja createGenericTransform jest funkcją wyższego rzędu, która przyjmuje funkcję transform (służącą do konwersji typu T na K) i zwraca nową funkcję. Ta nowa funkcja przyjmuje wartość typu T i zwraca wynik konwersji jako typ K.

Takie podejście pozwala nam wygodnie tworzyć różne funkcje transformujące, bazując na tym samym schemacie, i ponownie używać ich w różnych miejscach w kodzie.

Inny przykład to funkcja łącząca dwa obiekty:

function mergeObjects<T, U>(obj1: T, obj2: U): T & U {
    return { ...obj1, ...obj2 };
}

mergeObjects({
  firstName: 'John'
}, {
  lastName: 'Doe'
}); // { firstName: 'John', lastName: 'Doe' }

mergeObjects({
  id: 1,
  type: 'config'
}, {
  variant: 'regular'
}); // { id: 1, type: 'config', variant: 'regular' }

W tym przypadku typy generyczne określają dwa dowolne rodzaje obiektów, które na wyjściu połączymy (co zostało przedstawione jako przecięcie typów - operator &).

Klasy

Klasy, podobnie jak funkcje, również mogą zawierać odwołania do typów generycznych. Będą one wiązać określony typ na przestrzeni wybranych funkcji i pól w obrębie klasy:

class PersistedCollection<T> {

  async add(item: T) {
    // ...
  }

  async getAll(): T[] {
    // ...
  }

  async getById(id: string): T {
    // ...
  }
  
}

Taką generyczną klasę możemy wykorzystać np. w sytuacji, kiedy zestaw danych przechowujemy w bazie, a wprowadzona klasa PersistedCollection ma ukrywać detale połączenia z bazą, pozwalając nam na wykorzystanie różnych typów danych w ten sam sposób.

Interfejsy

Interfejsy, to trzeci obok klas i funkcji element języka TypeScript, w którym mogą znajdować się typy generyczne.

Popularnym zastosowaniem generycznych interfejsów są sytuacje, gdzie obok pól o ściśle zdefiniowanym kształcie (np. status i error o typach number i string) dodajemy jeszcze pola możliwe do zdefiniowania przez konsumenta (w tym przypadku - data):

interface ApiResponse<T> {
    data: T;
    status: number;
    error?: string;
}

Interfejs ApiResponse<T> wykorzystamy do zamodelowania odpowiedzi z API, gdzie obok statusu i ewentualnego kodu błędu, znajdzie się oczekiwana porcja danych (o type T).

Dzięki temu, w naszym projekcie możemy implementować funkcje pobierające dane z różnych domen, przy jednoczesnym zachowaniu spójności odpowiedzi:

function fetchUserData(id: number): ApiResponse<User> {
  ...
}

function fetchArticleData(id: number): ApiResponse<Article> {
  ...
}

Podsumowanie

Jeśli zauważysz w swoim kodzie powtórzoną logikę, identyczne funkcje lub zbieżne moduły, które różnią się wyłącznie typem przyjmowanego parametru, prawdopodobnie natrafiłeś na dobre miejsce do refaktoryzacji z użyciem typów generycznych.

Żeby przebiegła ona bezproblemowo, w kolejnych lekcjach pokażemy ci zaawansowane możliwości generyków a także przykłady ich wykorzystania w produkcyjnych bibliotekach Open Source.
